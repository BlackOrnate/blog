## 1. 查找的基本概念

- **查找表**：由同一类型的数据元素（或记录）构成的集合
- **关键字**：数据元素（或记录）中某个数据项的值，用它可以标识一个数据元素（或记录）
  - **主关键字**：可以唯一地标识一个记录的关键字
  - **次关键字**：用以识别若干记录的关键字
- **查找**：根据给定的某个值，在查找表中确定一个其关键字等于给定值的记录或数据元素
  - **查找成功**：表中存在这样一个记录，此时查找的结果可给出整个记录的信息，或指示该记录在查找表中的位置
  - **查找不成功**：表中不存在关键字等于给定值的记录，此时查找的结果可给出一个 “空” 记录或 “空” 指针
- **动态查找表**：在查找的同时又做修改操作（插入或删除）的表
- **静态查找表**：仅用于查找的表
- **平均查找长度（Average Search Length ASL）**：记录在查找表中的位置（和给定值进行比较的关键字数量的期望值）

## 2. 线性表的查找

### 2.1 顺序查找

- **顺序查找（Sequential Search）**：从表的一端开始，依次将记录的关键字和给定值进行比较
  - 若某个记录的关键字和给定值相等，则查找成功
  - 若没有记录的关键字和给定值相等，则查找失败

```c
void Search_Seq1(SqList L, int first) {
	for (int i = 0; i < L.length; i++)
	{
		if (L.abc[i].first == first)
		{
			printf("i = %d\n", i);
			printf("Search Success\n");
			return 0;
		}
	}
	printf("Search Error\n");
}
```

- 设置监视哨的顺序查找

```c
void Search_Seq2(SqList L, int first) {
	int i = 0;
	int value = L.abc[L.length - 1].first;
	L.abc[L.length-1].first = first;

	for (i = 0; L.abc[i].first != first; i++);

	L.abc[L.length - 1].first = value;
	printf("i = %d\n", i);
	printf("Search Success\n");
}
```

- 两种算法的时间复杂度相同  
   
  $$
  ASL = \frac{1}{n} \sum_{i=1}^{n}{i} = \frac{n+1}{2}
  $$
   时间复杂度为 O（n）
- 优点：算法简单，对表结构无任何要求，适用于顺序结构，也适用于链式结构，无论记录是否按关键字有序均可应用
- 缺点：平均查找长度较大，查找效率较低

### 2.2 折半查找

- **折半查找（Binary Search）（二分查找）**：线性表必须采用顺序存储结构，且表中元素按关键字有序排列
  - 从表的中间记录开始，如果给定值和中间记录的关键字相等，则查找成功
  - 如果给定值大于或小于中间记录的关键字，则在表中大于或小于中间记录的那一半中查找
  - 重复以上操作，直到查找成功，或在某一步中查找区域为空，则代表查找失败

```c
void Search_Bin(SqList L, int first) {
	int low = 0;
	int high = L.length - 1;
	int mid = 0;

	while (low <= high)
	{
		mid = (low + high) / 2;
		if (first == L.abc[mid].first)
		{
			printf("mid = %d\n", mid);
			printf("Search Success\n");
			return 0;
		}
		else if (first < L.abc[mid].first)
		{
			high = mid - 1;
		}
		else {
	
			low = mid + 1;
		}
	}
	printf("Search Error\n");
}
```

### 2.3 分块查找

- **分块查找（Blocking Search）（索引顺序查找）**：除了表本身外，引入一个 “索引表”

  - 将表中的记录分成子表，每个子表（块）建立一个索引
  - 索引内容
    - 关键字项（该子表内的最大关键字）
    - 指针项（该子表的第一个记录在表中位置）
  - 索引表按关键字有序，表或者有序或者分块有序
    - 分块有序：第二个子表中所有记录的关键字均大于第一个子表中的最大关键字，第三个子表中所有记录的关键字均大于第二个子表中的最大关键字，以此类推

  $$
  ASL_{bs} = L_b + L_w = \frac{1}{b} \sum_{j=1}^{b}{j} + \frac{1}{s} \sum_{i=1}^{s}{i} = \frac{b+1}{2} + \frac{s+1}{2} = \frac{1}{2} (\frac{n}{s} + s) + 1
  $$

## 3. 树表的查找

### 3.1 二叉排序树（后期补充）

- **二叉排序树（Binary Sort Tree）（二叉查找树）**：对排序和查找都很有用的特殊二叉树

![二叉排序树示例](https://i-blog.csdnimg.cn/blog_migrate/169e73f794e78e407eb046842a87b096.png#pic_center)

#### 3.1.1 定义

- 若二叉排序树的左子树不空，则左子树上所有结点的值均小于它的根结点的值
- 若二叉排序树的右子树不空，则右子树上所有结点的值均大于它的根结点的值
- 二叉排序树的左、右子树也分别为二叉排序树
- 中序遍历一棵二叉排序树可以得到一个节点值递增的有序序列

```c
typedef struct {
	int first;										// 关键字项
	int second;										// 其他数据项
} ABC;												// 每个结点的数据域的类型

typedef struct {
	ABC abc;										// 每个结点的数据域
	struct BSTNode* lchild, * rchild;				// 左右孩子指针
} BSTNode, *BSTree;
```

#### 3.1.2 查找

- 步骤
  - 若二叉排序树为空，则查找失败，返回空指针
  - 若二叉排序树非空，将给定值 key 与根结点的关键字 T->data.key 进行比较
    - 若 key 等于 T->data.key，则查找成功，返回根结点地址
    - 若 key 小于 T->data.key，则递归查找左子树
    - 若 key 大于 T->data.key，则递归查找右子树

```python
BSTree SearchBST(BSTree T, int value) {
	if ((!T) || value == T->abc.first)
	{
		return T;
	}
	else if (value < T->abc.first) {

		return SearchBST(T->lchild, value);
	}
	else {
		return SearchBST(T->rchild, value);
	}
}
```

#### 3.1.3 插入

- 步骤
  - 若二叉排序树为空，则待插入结点 \*S 作为根结点插入到空树中
  - 若二叉排序树非空，则将 key 与根结点的关键字 T->data.key 进行比较
    - 若 key 小于 T->data.key，则将 \*S 插入左子树
    - 若 key 大于 T->data.key，则将 \*S 插入右子树

```c
void InsertBST(BSTree* T, ABC abc) {
	if (!T)
	{
		BSTNode* S = NULL;
		S = (BSTNode*)malloc(sizeof(BSTNode));
		S->abc = abc;
		S->lchild = S->rchild = NULL;
		T = S;
	}
	else if (abc.first < (*T)->abc.first) {
		InsertBST((*T)->lchild, abc);
	}
	else if (abc.first > (*T)->abc.first) {
		InsertBST((*T)->rchild, abc);
	}
}
```

#### 3.1.4 创建

- 步骤
  - 将二叉排序树 T 初始化为空树
  - 读入一个关键字为 key 的结点
  - 如果读入的关键字 key 不是输入结束标志，则循环以下操作
    - 将此结点插入二叉排序树 T 中
    - 读入一个关键字为 key 的结点

```c
void CreatBST(BSTree* T) {
	int value = 0;
	ABC abc;
	while (value >= 0)
	{
		printf("输入结点: ");
		scanf("%d", &value);
		abc.first = abc.second = value;
		rewind(stdin);
		InsertBST(T, abc);
	}
	printf("Creat Success\n");
}
```

#### 3.1.5 删除

```c
int Delete(BiTree *p)
{
    BiTree q, s;
    if(!(*p)->lchild && !(*p)->rchild){
        *p = NULL;
    }
    else if(!(*p)->lchild){
        q = *p;
        *p = (*p)->rchild;
        free(q);
    }
    else if(!(*p)->rchild){
        q = *p;
        *p = (*p)->lchild;
        free(q);
    }
    else{
        q = *p;
        s = (*p)->lchild;
        while(s->rchild)
        {
            q = s;
            s = s->rchild;
        }
        (*p)->data = s->data;
        if( q != *p ){
            q->rchild = s->lchild;
        }else{
            q->lchild = s->lchild;
        }
        free(s);
    }
    return TRUE;
}

int DeleteBST(BiTree *T, int key)
{
    if( !(*T)){
        return FALSE;
    }
    else
    {
        if( key == (*T)->data ){
            Delete(T);
            return TRUE;
        }
        else if( key < (*T)->data){
            return DeleteBST(&(*T)->lchild, key);
        }
        else{
            return DeleteBST(&(*T)->rchild, key);
        }
    }
}
```

### 3.2 平衡二叉树（后期补充）

- **平衡二叉树（Balanced Binary Tree 或 Height-Balanced Tree）（AVL树）**：特殊类型的二叉排序树

![平衡二叉树的示例](https://i-blog.csdnimg.cn/blog_migrate/145702101c923091aa24f8bdb1941127.png#pic_center)

#### 3.2.1 定义

- 左子树和右子树的深度之差的绝对值不超过 1
- 左子树和右子树也是平衡二叉树
- 结点的**平衡因子（Balance Factor，BF）**：该结点左子树和右子树的深度之差

#### 3.2.2 平衡调整方法

- 找到离插入结点最近且平衡因子绝对值超过 1 的祖先结点，以该结点为根的子树称为**最小不平衡子树**，可将重新平衡的范围局限于这棵子树
- 假设最小不平衡子树的根结点为 A ，则失去平衡后进行调整的规律有以下 4 种情况

  - LL 型：在 A 左子树根结点的左子树上插入结点，A 的平衡因子由 1 变为 2 ，致使以 A 为根的子树失去平衡，则需进行一次向右的顺时针旋转操作

    ![LL型调整操作示意图](https://i-blog.csdnimg.cn/blog_migrate/f4f954e2ee595416eb303cafdd1885f6.png#pic_center)
  - RR 型：在 A 右子树根结点的右子树上插入结点，A 的平衡因子由 -1 变为 -2 ，致使以 A 为根的子树失去平衡，则需进行一次向左的逆时针旋转操作

    ![RR型调整操作示意图](https://i-blog.csdnimg.cn/blog_migrate/256d4fe79db5d91935a838f92a1261af.png#pic_center)
  - LR 型：在 A 的左子树根结点的右子树上插入结点，A 的平衡因子由 1 变为 2 ，致使以 A 为根的子树失去平衡，则需进行两次旋转操作

    - 对 B 及其右子树进行逆时针旋转，C 转上去成为 B 的根，此时变成 LL 型
    - 进行 LL 型的顺时针旋转

    ![LR(L)型调整操作示意图](https://i-blog.csdnimg.cn/blog_migrate/e58ff932696e1dfec4de6ae735028959.png#pic_center)

    ![LR(R)型调整操作示意图](https://i-blog.csdnimg.cn/blog_migrate/33f44cb16b2dabe4af6e4dffbd3888a9.png#pic_center)
  - RL 型：在 A 的右子树根结点的左子树上插入结点，A 的平衡因子由 -1 变为 -2 ，致使以 A 为根的子树失去平衡，则需进行两次旋转操作

    - 对 B 及其左子树进行顺时针旋转，C 转上去成为 B 的根，此时变成 RR 型
    - 进行 RR 型的逆时针旋转

### 3.3 B-树（后期补充）

- B-树：适用于外查找的平衡多叉树

#### 3.3.1 定义

- 一棵 m 阶的 B-树，m 叉树

  - 树中每个结点至多有 m 棵子树
  - 若根结点不是叶子结点，则至少有两棵子树
  - 除根之外的所有非终端结点至少有 ⌈m/2⌉ 棵子树
  - 所有的叶子结点都出现在同一层上，并且不带信息，通常称为失败结点
    - 失败结点并不存在，指向这些结点的指针为空
    - 引入失败结点是为了便于分析 B-树 的查找性能
  - 所有的非终端结点最多有 m-1 个关键字
- 特点

  - 平衡：所有的叶子结点均在同一层次
  - 有序：树中每个结点中的关键字都是有序的，且关键字 K\_i “左子树” 中的关键字均小于 K\_i ，而其 “右子树” 中的关键字均大于 K\_i
  - 多路：除叶子结点外，有的结点中有一个关键字，两棵子树，有的结点中有两个关键字，三棵子树

### 3.4 B+树

- B+ 树是一种 B-树 的变形树

#### 3.4.1 B+树 和 B-树 的差异

- 有 n 棵子树的结点中含有 n 个关键字
- 所有的叶子结点中包含了全部关键字的信息，以及指向含这些关键字记录的指针，且叶子结点本身依关键字的大小自小而大顺序链接
- 所有的非终端结点可以看成是索引部分，结点中仅含有其子树（根结点）中的最大（或最小）关键字

#### 3.4.2 查找

- 若非终端结点上的关键字等于给定值，并不终止，而是继续向下直到叶子结点

#### 3.4.3 插入

- 仅在叶子结点上进行插入，当结点中的关键字个数大于 m 时要分裂成两个结点，它们所含关键字的个数分别为 ⌊(m+1)/2⌋ 和 ⌈(m+1)/2⌉
- 它们的双亲结点中应同时包含这两个结点中的最大关键字

#### 3.4.4 删除

- B+树 的删除也仅在叶子结点进行，当叶子结点中最大关键字被删除时，其在非终端结点中的值可以作为一个 “分界关键字” 存在
- 若因删除而使结点中关键字的个数少于 ⌈m/2⌉ 时，其和兄弟结点的合并过程和 B-树 类似

## 4. 散列表的查找

### 4.1 概念

- **散列查找（Hash Search）（杂凑，散列）**：通过对元素的关键字值进行某种运算，直接求出元素的地址，即使用关键字到地址的直接转换方法，不需要反复比较
- **散列地址**：记录的存储位置
- **散列函数**：记录的存储位置和其关键字之间建立的一个确定的对应关系
- **散列表**：一个有限连续的地址空间，用以存储按散列函数计算得到相应散列地址的数据记录

  - 通常**散列表**的存储空间是一个一维数组，散列地址是数组的下标
- **冲突**：对不同的关键字得到同一散列地址
- **同义词**：具有相同函数值的关键字

### 4.2 构造方法

#### 4.2.1 数字分析法

- 事先知道关键字集合，且每个关键字的位数比散列表的地址码位数多，每个关键字由 n 位数组成，如 k\_1 k\_2 ··· k\_n ，则可以从关键字中提取数字分布比较均匀的若干位作为散列地址
- 适用情况：事先必须明确知道所有的关键字每一位上各种数字的分布情况

#### 4.2.2 平方取中法

- 一个数平方后的中间几位数和数的每一位都相关，如果取关键字平方后的中间几位或其组合合作为散列地址，则使随机分布的关键字得到的散列地址也是随机的，具体所取得位数由表长决定
- 适用情况：不能事先了解关键字得所有情况，或难于直接从关键字中找到取值较分散的几位

#### 4.2.3 折叠法

- 将关键字分割成位数相同的几部分（最后一部分的位数可以不同），然后取这几部分的叠加和（舍去进位）作为散列地址

  - 位移叠加：将分割后每一部分的最低位对齐，然后相加
  - 边界叠加：将两个相邻的部分沿边界来回折叠，然后对齐相加
- 适用情况：散列地址的位数较少，而关键字的位数较多，且难于直接从关键字中找到取值较分散的几位

#### 4.2.4 除留余数法

- 假设散列表表长为 m ，选择一个不大于 m 的数 p ，用 p 去除关键字，除后所得余数为散列地址
- 能够保证散列地址一定落在散列表的地址空间中

### 4.3 处理冲突的方法

#### 4.3.1 开放地址法

- 基本思想：

  - 把记录存储在散列表中，当某一记录关键字 key 的初始散列地址 H\_0 = H(key) 发生冲突时，以 H\_0 为基础，采取合适方法计算得到另一个地址 H\_1 ，如果 H\_1 仍然发生冲突，以 H\_1 为基础再求下一个地址 H\_2 ，若 H\_2 仍然冲突，再求得 H\_3 ，依此类推，直至 H\_k 不发生冲突为止，则 H\_k 为该记录在表中的散列地址
- 开放地址法：在寻找 “下一个” 空的散列地址时，原来的数组空间对所有的元素都是开放的
- **探测**：寻找 “下一个” 空位的过程  
   
  $$
  H_i = (H(key) + d_i) \% m \quad i = 1,2,···,k (k \leq m-1)
  $$
   H(key) ：散列函数 ；m ：散列表表长 ，d\_i ：增量序列

### 4.4 查找

- 散列表的存储

```c
#define m 10										// 散列表的表长

typedef struct {
    KeyType key;									// 关键字项
    InfoType otherinfo;								// 其他数据项
} HashTable[m];
```

- 步骤
  - 给定待查找的关键字 key ，根据造表时设定的散列函数计算 H\_0 = H(key)
  - 若单元 H\_0 为空，则所查元素不存在
  - 若单元 H\_0 中元素的关键字为 key ，则查找成功
  - 否则重复下述解决冲突的过程
    - 按处理冲突的方法，计算下一个散列地址 H\_i
    - 若单元 H\_i 为空，则所查元素不存在
    - 若单元 H\_i 中元素的关键字为 key ，则查找成功

```c
void SearchHash(HashTable HT, KeyType key) {
    int H_0 = H(key);
    for (int i = 0; i < m; i++) {
        int H_i = (H_0 + i) % m;
        if (HT[H_i].key == 0) {
            printf("Not Exist");
            return 0;
        } else if (HT[H_i].key == key){
            printf("Search Success");
            return 0;
        }
    }
}
```

- 算法分析

  - 散列表的**装填因子α**定义为  
     
    $$
    \alpha = \frac{表中填入的记录数}{散列表的长度}
    $$
  - 等概率情况下，不同方法处理冲突时散列表的平均查找长度

    - 线性探测法

      - 查找成功  
         
        $$
        \frac{1}{2} (1 + \frac{1}{1-\alpha})
        $$
      - 查找失败  
         
        $$
        \frac{1}{2} [1 + \frac{1}{(1-\alpha)^2}]
        $$
    - 二次探测法，伪随机探测法

      - 查找成功  
         
        $$
        -\frac{1}{\alpha} ln(1-\alpha)
        $$
      - 查找失败  
         
        $$
        \frac{1}{1-\alpha}
        $$
    - 链地址法

      - 查找成功  
         
        $$
        1 + \frac{\alpha}{2}
        $$
      - 查找失败  
         
        $$
        \alpha + e^{- \alpha}
        $$
        α+e−α
  - 等概率情况下，直接计算平均查找长度

    - 查找成功  
       
      $$
      ASL_{succ} = \frac{1}{n} \sum_{i=1}^{n}{C_i}
      $$
       n ：散列表中记录的个数 ；C\_i ：成功查找第 i 个记录所需的比较次数
    - 查找失败  
       
      $$
      ASL_{unsucc} = \frac{1}{r} \sum_{i=1}^{r}{C_i}
      $$
       n ：散列函数取值的个数 ；C\_i ：散列函数取值为 i 时查找失败的比较次数
