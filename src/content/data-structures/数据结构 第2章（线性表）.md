## 1. 线性表的定义和特点

- **线性表**：由 n（n ≥ 0）个数据特性相同的元素构成的有限序列

  - 线性表的长度：元素的个数 n
  - **空表**：n = 0 时
- 对于非空的线性表或线性结构，特点：

  - 存在唯一的一个被称作 “第一个” 的数据元素
  - 存在唯一的一个被称作 “最后一个” 的数据元素
  - 除第一个之外，结构中的每个数据元素均只有一个前驱
  - 除最后一个之外，结构中的每个数据元素均只有一个后继

## 2. 线性表的顺序表示和实现

### 2.1 线性表的顺序存储表示

- **顺序表（Sequential List）**：用一组地址连续的存储单元依次存储线性表的数据元素

  - 也称为线性表的顺序存储结构或顺序映像
  - 特点：
    - 逻辑上相邻的数据元素
    - 物理次序也是相邻的
- 假设线性表的每个元素需占用 l 个存储单元，并以所占的第一个单元的存储地址作为数据元素的存储起始位置

  - 第 i + 1 个数据元素的存储位置 LOC(ai+1) 和 第 i 个数据元素的存储位置 LOC(ai) 满足如下关系  
    
    $$
    LOC(a_{i+1}) = LOC(a_i) + l
    $$
    
  - 线性表的第 i 个元素 ai 的存储位置为  
    
    $$
    LOC(a_i) = LOC(a_1) + (i-1)*l
    $$
    
    
    - LOC(ai) 通常称作线性表的起始位置或基地址
  - 只要确定了存储线性表的起始位置，线性表中任一数据元素都可随机存储，所以线性表的顺序存储结构是一种**随机存储**的存储结构
- 通常使用数组来描述数据结构中的顺序存储结构

  - 由于线性表的长度可变，而且所需最大存储空间随问题不同而不同，则在 C 语言中用动态分配的一维数组表示线性表

    ```c
    #define MAXSIZE 5				// 线性表可能达到的最大长度
    
    typedef struct {
        ElemType *elem;				// 存储空间的基地址
        int length;					// 当前长度
    } SqList;						// 顺序表的结构类型为 SqList
    ```

    - ElemType 是为了统一描述自定的，实际应用中，用户根据实际需要具体定义表中数据元素的数据类型
      - 可以是基本数据类型
        - int 、float 、char 等
      - 也可以是构造数据类型
        - struct 结构体类型
- 表示多项式的顺序存储示例  
  
  $$
  P_n(x) = p_1x^{e_1} + p_2x^{e_2} + \cdots + p_mx^{e_m}
  $$

  $$
  0 \leq e_1 < e_2 < \cdots < e_m = n
  $$

  - 用一个长度为 m 且每个元素有两个数据项（系数项和指数项）的线性表

  $$
  [(p_1,e_1),(p_2,e_2),\cdots,(p_m,e_m)]
  $$

  ```c
  #define MAXSIZE 5					// 线性表可能达到的最大长度
  
  typedef struct {					// 多项式的定义
      float coef;						// 系数
      int expn;						// 指数
  } Polynomial;
  
  typedef struct {
      Polynomial *elem;				// 存储空间的基地址
      int length;						// 多项式中当前项的个数
  } SqList;							// 多项式的顺序存储结构类型为 SqList
  ```

### 2.2 顺序表中基本操作的实现

#### 2.2.1 初始化

- 构造一个空的顺序表
- 步骤：

  - 为顺序表动态分配一个预定大小的数组空间，使 elem 指向这段空间的基地址
  - 将表的当前长度设为 0

```c
void InitList(SqList* L) {
    L->abc = (ABC *) malloc(MAXSIZE * sizeof(ABC));
    L->length = 0;
    printf("Init Success\n");
}
```

#### 2.2.2 取值

- 根据指定的位置序号 i ，获取顺序表中第 i 个数据元素的值

```c
void GetElem(SqList* L, int i) {
    printf("Get Success\n");
    printf("first = %d\n", L->abc[i].first);
    printf("second = %d\n", L->abc[i].second);
}
```

#### 2.2.3 查找

- 从第一个元素起依次比较，获取比较成功的数据

```c
void LocateElem(SqList* L, int value) {
    printf("Locate Success\n");
    int j;
    for (int i = 0; i < L->length; i++)
    {
        j = L->abc[i].second;
        if (j == value)
        {
            printf("index = %d\n", i);
        }
    }
}
```

- 在查找时，为确定元素在顺序表中的位置，需和给定值进行比较的数据元素个数的期望值称为查找算法在查找成功时的**平均查找长度（Average Search Length，ASL）**
- 假设 pi 是查找第 i 个元素的概率，Ci 为找到表中其关键字与给定值相等的第 i 个记录时，和给定值已进行过比较的关键字个数，则在长度为 n 的线性表中，查找成功时的平均查找长度为  
  
  $$
  ASL = \sum_{i=1}^{n}{p_ic_i}
  $$
   一般情况下 Ci = i ，pi = 1/n，所以 ASL 可以简化为  
  
  $$
  ASL = \frac{1}{n} \sum_{i=1}^{n}{i} = \frac{n+1}{2}
  $$
   因此，顺序表按值查找算法的平均时间复杂度为 O(n)

#### 2.2.4 插入

- 在表的第 i 个位置插入一个新的数据元素 e ，使长度为 n 的线性表  
  
  $$
  (a_1,\cdots,a_{i-1},a_i,\cdots,a_n)
  $$
   变成长度为 n+1 的线性表  
  
  $$
  (a_1,\cdots,a_{i-1},e,a_i,\cdots,a_n)
  $$

  - 数据元素 ai-1 和 ai 之间的逻辑关系发生了变化
  - 在顺序表中，由于逻辑上相邻的数据元素在物理位置上也是相邻的，所以，除非 i = n+1 ，否则必须移动元素才能反映这个逻辑关系的变化
  - 一般情况下，在第 i （1 ≤ i ≤ n）个位置插入一个元素时，需从最后一个元素即第 n 个元素开始，依次向后移动一个位置，直至第 i 个元素（共 n-i+1 个元素）
- 步骤：

  - 判断插入位置 i 是否合法
  - 判断顺序表的存储空间是否已满
  - 将第 n 个至第 i 个位置的元素依次向后移动一个位置，空出第 i 个位置
  - 将新元素 e 放入第 i 个位置
  - 表长加1

```c
void ListInsert(SqList* L, ABC abc, int index) {
    if (index < 0 || index > L->length || L->length == MAXSIZE)
    {
        printf("Insert Error\n");
        return 0;
    }

    int i = L->length - 1;

    while ((i - index) != -1)
    {
        L->abc[i + 1] = L->abc[i];
        i--;
    }

    L->abc[index] = abc;
    L->length += 1;

    printf("Insert Success\n");
}
```

![顺序表插入](https://i-blog.csdnimg.cn/blog_migrate/501a3ff17fc1d8594d2fd78f19c7bdcc.png#pic_center)

- 假设 pi 是在第 i 个元素之前插入一个元素的概率，Eins 为在长度为 n 的线性表中插入一个元素时所需移动元素次数的期望值（平均次数），则有  
  
  $$
  E_{ins} = \sum_{i=1}^{n+1}{p_i(n-i+1)}
  $$
   一般情况下，pi = 1/(n+1)，所以 Eins 可以简化为  
  
  $$
  E_{ins} = \frac{1}{n+1} \sum_{i=1}^{n+1}{(n-i+1)} = \frac{n}{2}
  $$
   因此，顺序表插入算法的平均时间复杂度为 O(n)

#### 2.2.5 删除

- 将表的第 i 个元素删去，使长度为 n 的线性表  
  
  $$
  (a_1,\cdots,a_{i-1},a_i,a_{i+1}\cdots,a_n)
  $$
   变成长度为 n-1 的线性表  
  
  $$
  (a_1,\cdots,a_{i-1},a_{i+1},\cdots,a_n)
  $$

  - 数据元素 ai-1、 ai 和 ai+1 之间的逻辑关系发生了变化，为了在存储结构上反映这个变化，同样需要移动元素
  - 一般情况下，删除第 i （1 ≤ i ≤ n）个元素时需将第 i+1 个至第 n 个元素（共 n-i 个元素）依次向前移动一个位置
- 步骤：

  - 判断删除位置 i 是否合法
  - 将第 i+1 个至第 n 个位置的元素依次向前移动一个位置
  - 表长减1

```c
void ListDelete(SqList* L, int index) {
    int i = L->length;

    if (index < 0 || index > (L->length - 1))
    {
        printf("Delete Error\n");
        return 0;
    }

    while ((i - 1 - index) != -1)
    {
        L->abc[index] = L->abc[index+1];
        index++;
    }

    L->length -= 1;

    printf("Delete Success\n");
}
```

![顺序表删除](https://i-blog.csdnimg.cn/blog_migrate/0fc58bfab0ad9cb069931aab77d83a4d.png#pic_center)

- 假设 pi 是删除第 i 个元素的概率，Edel 为在长度为 n 的线性表中删除一个元素时所需移动元素次数的期望值（平均次数），则有  
  
  $$
  E_{del} = \sum_{i=1}^{n}{p_i(n-i)}
  $$
   一般情况下，pi = 1/n，所以 Edel 可以简化为  
  
  $$
  E_{del} = \frac{1}{n} \sum_{i=1}^{n}{(n-i)} = \frac{n-1}{2}
  $$
   因此，顺序表插入算法的平均时间复杂度为 O(n)

### 顺序表总结

- 顺序表可以随机存取表中任一元素，其存储位置可用一个简单、直观的公式来表示
- 然而，这个特点也造成了这种存储结构的缺点：
  - 在做插入或删除出操作时，需移动大量元素
- 由于数组有长度相对固定的静态特性，当表中数据元素个数较多且变化较大时，操作过程相对复杂，必然导致存储空间的浪费

## 3. 线性表的链式表示和实现

### 3.1 单链表的定义和表示

- **链表**：用一组任意的存储单元存储线性表的数据元素（这组存储单元可以是连续的，也可以是不连续的）

  - 为了表示每个数据元素 ai 与其直接后继数据元素 ai+1 之间的逻辑关系，对数据元素 ai 来说，除了存储其本身的信息之外，还需存储一个指示其直接后继的信息（直接后继的存储位置）
  - **结点（node）**：以上两个部分信息组成数据元素 ai 的存储映像
    - 包含两个域
      - **数据域**：存储数据元素信息的域
      - **指针域**：存储直接后继存储位置的域
        - 存储的信息称作**指针**或**链**
  - n 个结点链结成一个链表，即为线性表的链式存储结构
    - 由于此链表中的每个结点中只包含一个指针域，故又称**线性链表**或**单链表**
- 单链表：整个链表的存取必须从头指针开始进行，头指针指示链表第一个结点的存储位置

  - 由于最后一个数据元素没有直接后继，则最后一个结点的指针为空（NULL）

```c
typedef struct LNode {
	ElemType data;					// 结点的数据域
	struct LNode* next;				// 结点的指针域
}LNode, *LinkList;					// LinkList 为指向结构体 LNode 的指针类型
```

- 一般情况下，在单链表的第一个结点之前附设一个结点，称之为**头结点**
- 添加头结点的作用

  - 便于首元结点的处理
  - 便于空表和非空表的统一处理

### 3.2 单链表基本操作的实现

#### 3.2.1 初始化

- 步骤
  - 生成新结点作为头结点，用头结点 L 指向头结点
  - 头结点的指针域置空

```c
void InitList(LinkList* L, ABC abc) {
	*L = (LinkList)malloc(sizeof(LNode));
	(*L)->abc = abc;
	(*L)->next = NULL;
	printf("Init Success\n");
}
```

#### 3.2.2 取值

- 步骤
  - 用指针 p 指向首元结点，用 j 做计数器初值赋为 1
  - 从首元结点开始依次顺着链域 next 向下访问，只要指向当前结点的指针 p 不为空，且没有到达序号为 index 的结点，循环执行
    - p 指向下一个结点
    - 计数器 i 加 1

```c
void GetElem(LinkList* L, int index, ABC* abc) {
	LNode* p = (*L)->next;
	int i = 0;

	if (index >= (*L)->length)
	{
		printf("Get Error\n");
		return 0;
	}

	while (i < index)
	{
		i++;
		p = p->next;
	}

	*abc = p->abc;

	printf("Get Success\n");
}
```

- 比较 i 和 index 并后移指针 p ， while 循环体中的语句频度与位置 i 有关

  - 若 1 ≤ i ≤ n，则频度为 i-1 ，一定能取值成功
  - 若 i > n ，则频度为 n ，取值失败
  - 最坏时间复杂度为 O(n)

  一般情况下，pi = 1/n ，则  
  
  $$
  ASL = \frac{1}{n} \sum_{i=1}^{n}{(i-1)} = \frac{n-1}{2}
  $$
   因此，单链表取值算法的平均时间复杂度为 O(n)

#### 3.2.3 查找

- 步骤
  - 用指针 p 指向首元结点
  - 从首元结点开始依次顺着链域 next 向下查找，只要指向当前结点的指针 p 不为空，且没有到达序号为 index 的结点，循环执行
    - p 指向下一个结点

```c
void LocateElem(LinkList* L, ABC abc) {
	LNode* p = (*L)->next;

	while (p != NULL)
	{

		if (p->abc.first == abc.first && p->abc.second == abc.second)
		{
			printf("p->abc.first = %d\n", abc.first);
			printf("p->abc.second = %d\n", abc.second);
			return 0;
		}

		p = p->next;

	}
	printf("Locate Error\n");
	return 0;
}
```

- 单链表查找算法的平均时间复杂度为 O(n)

#### 3.2.4 插入

- 假设要在单链表的两个数据元素 a 和 b 之间插入一个数据元素 x

  - 生成一个数据域为 x 的结点，然后插入到单链表中
  - 修改结点 a 中的指针域，令其指向结点 x ，而结点 x 中的指针应指向结点 b，从而实现 3 个元素 a 、b 、x 之间的逻辑关系变化
- 步骤

  - 查找结点 ai-1 并由指针 p 指向该结点
  - 生成新结点 \*s
  - 将新结点 \*s 的数据域置为 e
  - 将新结点 \*s 的指针域指向结点 ai
  - 将结点 \*p 的指针域指向新结点 \*s

```c
void ListInsert(LinkList* L, int index, ABC abc) {
	LinkList new = (LinkList)malloc(sizeof(LNode));

	new->abc = abc;
	new->next = NULL;

	LNode* p = (*L)->next;
	int i = 0;

	if (index > (*L)->length)
	{
		printf("Insert Error\n");
		return 0;
	}

	while (i < (index - 1))
	{
		i++;
		p = p->next;
	}

	new->next = p->next;
	p->next = new;

	(*L)->length++;

	printf("Insert Success\n");
}
```

![链表插入](https://i-blog.csdnimg.cn/blog_migrate/9faa94ffffa959ea3d9016ee7136b7e4.png#pic_center)

- 单链表插入算法的平均时间复杂度为 O(n)

#### 3.2.5 删除

- 步骤
  - 查找结点 ai-1 并由指针 p 指向该结点
  - 临时保存待删除结点 ai 的地址在 q 中，以备释放
  - 将结点 \*p 的指针域指向 ai 的直接后继结点
  - 释放结点 ai 的空间


#### 3.2.6 创建单链表

- 链表的创建方法可分为头插法和尾插法

##### (1) 头插法

- 将新结点逐个插入链表的头部（头结点之后）来创建表，每次申请一个新结点，读入相应的数据元素，然后将新结点插入到头结点之后
- 步骤

  - 创建一个只有头结点的空链表
  - 根据待创建链表包括的元素个数 n ，循环 n 次执行以下操作
    - 生成一个新结点 \*p
    - 输入元素值赋给新结点 \*p 的数据域
    - 将新结点 \*p 插入到头结点之后

```c
void CreateList_H(LinkList* L, int num) {
	*L = (LinkList)malloc(sizeof(LNode));
	(*L)->abc.first = 0;
	(*L)->abc.second = 0;
	(*L)->next = NULL;
	(*L)->length = 0;

	for (int i = num; i > 0; i--)
	{
		LinkList p;
		p = (LinkList)malloc(sizeof(LNode));
		p->abc.first = i;
		p->abc.second = i;

		p->next = (*L)->next;
		(*L)->next = p;
	}
}
```

![头插法创建单链表](https://i-blog.csdnimg.cn/blog_migrate/ff830ebbad3ca3e7969ee50bb235a675.png#pic_center)

- 单链表头插法算法的平均时间复杂度为 O(n)

##### (2) 尾插法

- 将新结点逐个插入到链表的尾部来创建链表
- 每次申请一个新结点，读入相应的数据元素
- 为了使新结点能够插入到表尾，需要增加一个尾指针 r 指向链表的尾结点
- 步骤
  - 创建一个只有头结点的空链表
  - 尾指针 r 初始化，指向头结点
  - 根据待创建链表包括的元素个数 n ，循环 n 次执行以下操作
    - 生成一个新结点 \*p
    - 输入元素值赋给新结点 \*p 的数据域
    - 将新结点 \*p 插入到尾结点 \*r 之后
    - 尾指针 r 指向新的尾结点 \*p

```c
void CreateList_R(LinkList* L, int num) {
	*L = (LinkList)malloc(sizeof(LNode));
	(*L)->abc.first = 0;
	(*L)->abc.second = 0;
	(*L)->next = NULL;
	(*L)->length = 0;

	LinkList r = *L;

	for (int i = 0; i < num; i++)
	{
		LinkList p;
		p = (LinkList)malloc(sizeof(LNode));
		p->abc.first = i + 1;
		p->abc.second = i + 1;

		p->next = NULL;
		r->next = p;
		r = p;
	}
}
```

- 单链表尾插法算法的平均时间复杂度为 O(n)

### 3.3 循环链表

- **循环链表（Circular Linked List）**：最后一个结点的指针域指向头结点，整个链表形成一个环

  - 从表中任一结点除法均可找到表中其他的结点
- 单链表和循环链表的差别：链表遍历时，判别当前指针 p 是否指向表尾结点的终止条件不同

  - 单链表
    - p != NULL 或 p -> next != NULL
  - 循环链表
    - p != L 或 p -> next != L
- 在某些情况下，在循环链表中设立尾指针而不设头指针，可使一些操作简化

  - 将两个表合并

    - 将第一个表的尾指针指向第二个表的第一个结点，第二表的尾指针指向第一个表的头结点，释放第二表的头结点

    ```c
    // 将表 A 和 B 合并
    p = B->next->next;
    B->next = A->next;
    A->next = p;
    ```

### 3.4 双向链表

- 在单链表中，查找直接后继结点的执行时间为 O(1) ，查找直接前驱的执行时间为 O(n)
- 为克服这种单向性的缺点，可使用**双向链表（Double Linked List）**
  - 双向链表的结点中有两个指针域，一个指向直接后继，另一个指向直接前驱

```c
typedef struct DuLNode {
    ElemType data;					// 数据域
    struct DuLNode *prior;			// 指向直接前驱
    struct DuLNode *next;			// 指向直接后继
} DuLNode, *DuLinkList;
```

- 双向链表也可以有循环表

## 4. 顺序表和链表的比较

### 4.1 空间性能的比较

- 存储空间的分配

  - 顺序表：存储空间必须预先分配，元素个数扩充受一定限制，易造成存储空间浪费或空间溢出现象
  - 链表：只要内存空间允许，链表中的元素个数就没有限制
- 当线性表的长度变化较大，难以预估存储规模时，宜采用链表作为存储结构
- 存储密度的大小

  - **存储密度**：数据元素本身占用的存储量和整个结点结构所占用的存储量之比  
    
    $$
    存储密度=\frac{数据元素本身占用的存储量}{结点结构占用的存储量}
    $$
     存储密度越大，存储空间占用率越高
  - 链表的每个结点除了设置数据域用来存储数据元素外，还要额外设置指针域，用来存储指示元素之间逻辑关系的指针，从存储密度上来讲，这是不经济的
  - 顺序表的存储密度为 1 ，而链表的存储密度小于 1
- 当线性表的长度变化不大，易于事先确定其大小时，为了节约存储空间，宜采用顺序表作为存储结构

### 4.2 时间性能的比较

- 存取元素的效率

  - 顺序表：由数组实现，是一种随机存取结构，指定任意一个位置序号 i ，都可以在 O(1) 时间内直接存取该位置上的元素
    - 取值操作的效率高
  - 链表：是一种顺序存取结构，按位置访问链表中第 i 个元素时，只能从表头开始依次向后遍历链表，直到找到第 i 个位置上的元素，时间复杂度为 O(n)
    - 取值操作的效率低
- 若线性表的主要操作是和元素位置紧密相关的这类操作，很少做插入或删除时，宜采用顺序表作为存储结构
- 插入和删除操作的效率

  - 顺序表：平均要移动表中近一半的结点，时间复杂度为 O(n)
  - 链表：无需移动数据，只需要修改指针，时间复杂度为 O(1)
- 对于频繁进行插入或删除操作的线性表，宜采用链表作为存储结构
