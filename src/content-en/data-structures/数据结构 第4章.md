## 1. Definition of a string

- **String** (string) (or **character string**): a finite sequence of zero or more characters, notated as s = "a\_1 a\_2 --- a\_n" (n ≥ 0)

  - s : the name of the string, the sequence of characters enclosed in double quotes is the value of the string
  - a\_i (1 ≤ i ≤ n) : can be letters, numbers or other characters
  - n: the string of ** length **
  - **null string (null string)**: string of zero characters, length zero
- **Substring**: a subsequence of any consecutive characters in the string
- **main string**: the string containing the substring
- **position**: the number of the character in the sequence

  - Position of the substring in the main string: expressed as the position of the first character of the substring in the main string

    ```
    a = "abcd";
    b = "ABCD";
    c = "abcdABCD";
    d = "abcd ABCD";

    // a 和 b 都是 c 和 d 的子串
    // a 在 c 和 d 中的位置都是 1
    // b 在 c 中的位置是 5
    // b 在 d 中的位置是 6
    ```.
- **Equal**: two strings have equal values

  - The lengths of the two strings are equal
  - The characters in each corresponding position are equal
- **blank string**: a string consisting of one or more blanks.

  - Not a blank string.
  - The length is the number of blank characters in the string.
  - The symbol "∅" is used to indicate an "empty string".

### 2.2 Pattern matching algorithm for strings

- **Pattern matching** / **String matching**: substring localization operations
- Addendum: The following code all uses fixed-length sequential storage structures

  - Not recommended, too cumbersome
  - Define the string directly and use strlen() to get the length of the string.

#### 2.2.1 BF Algorithm

- **BF: Brute - Force**
- Steps
  - Use the counting pointers i and j to indicate the current position of the character to be compared in the main string S and the pattern (substring) T. The initial value of i is pos and the initial value of j is 1.
  - If i and j are both less than or equal to the length of S and T respectively, then the following operations are performed in a loop
    - S.ch[i] and T.ch[j] comparison
      - Equal: i and j indicate the next position in the string, and the comparison continues with the next characters.
      - unequal: the pointer backs up and starts the match again, starting with the next character in the main string and comparing again with the first character of the pattern.

```c
void Index_BF(SString S, SString T, int index) {
	int i = index;
	int j = 0;

	while (i < S.length && j < T.length )
	{
		if (S.ch[i] == T.ch[j])
		{
			i++;
			j++;
		}
		else {
			i = ++index;
			j = 0;
		}
	}

	if (j > T.length - 1)
	{
		printf("Index_BF Success\n");
		printf("Index from %d to %d\n", index, index + j - 1);
	}
	else
	{
		printf("Error\n");
	}
}
```

! [Matching process of BF algorithm](https://i-blog.csdnimg.cn/blog_migrate/804bda3d1adec44538fe35b16a2fa921.png#pic_center)

- Best case: each unsuccessful match occurs when the first character of the pattern string is compared to the corresponding character in the main string

  ```
  S = "aaaaaba";
  T = "ba";
  ```
- Let the length of the main string be n , the length of the substring be m , and the probability of successful matching is equal everywhere, then the average number of comparisons is

  $$
  \sum_{i=1}^{n-m+1}{p_n(i-1+m)} = \frac{1}{n-m+1}\sum_{i=1}^{n-m+1}{i-1+m} = \frac{1}{2}(n+m)
  $$
   Therefore, the average time complexity in the best case is O(n+m)

#### 2.2.2 KMP algorithm (unreadable)

- Instead of backtracking to the i-pointer whenever there are unequal character comparisons in a matching trip, use the results of the already obtained "partial matches" to "slide" the pattern as far to the right as possible before continuing the comparison.

```c
void Index_KMP(SString S, SString T) {
	int next[10];

	// 根据模式串 T，初始化 next 数组
	Next(T, next);

	int i = 1;
	int j = 1;

	while (i <= S.length && j <= T.length)
	{
		// j==0 ：模式串的第一个字符就和当前测试的字符不相等
		// S[i-1] == T.ch[j-1] ：如果对应位置字符相等，两种情况下，指向当前测试的两个指针下标i和j都向后移
		if (j == 0 || S.ch[i-1] == T.ch[j-1])
		{
			i++;
			j++;
		}
		else
		{
			// 如果测试的两个字符不相等，i不动，j变为当前测试字符串的next值
			j = next[j];
		}
	}

	if (j > T.length)
	{
		printf("Success\n");

	}
	else
	{
		printf("Error\n");
	}
}

void Next(SString T, int* next) {
	int i = 1;
	next[1] = 0;
	int j = 0;

	while (i < T.length)
	{
		if (j == 0 || T.ch[i-1] == T.ch[j-1])
		{
			i++;
			j++;
			next[i] = j;
		}
		else
		{
			j = next[j];
		}
	}
}
```

### 3.3 Compressed storage of special matrices

#### 3.3.1 Symmetric matrices

- a\_ij = a\_ji , 1 ≤ i, j ≤ n
- Store the elements in the lower triangles (including diagonals)
- Suppose sa [n(n+1)/2] is used as the storage structure of the nth order symmetric matrix A
- The correspondence between sa [k] and the matrix element a\_ij is

  $$
  k=\begin{cases} \frac{i(i-1)}{2} + j - 1 \quad 当 i ≥ j \\ \frac{j(j-1)}{2} + i - 1 \quad 当 i < j\end{cases}
  $$

$$
  k=\begin{cases} \frac{i(i-1)}{2} + j - 1 \quad 当 i ≥ j \\ \frac{j(j-1)}{2} + i - 1 \quad 当 i < j\end{cases}
  $$! [Compressed storage of symmetric matrices](https://i-blog.csdnimg.cn/blog_migrate/7da4948df9e4316f226ba1d443868041.png#pic_center)

- Derivation process

  - The symmetric matrix has a total number of elements of

    $$
    \frac{n*(n+1)}{2}
    $$
  - The range of subscripts is

    $$
    [0, \frac{n*(n+1)}{2} - 1]
    $$
     The subscript of a\_nnn is

    $$
    \frac{n*(n+1)}{2} - 1
    $$
     The subscript of a\_n1 is

    $$
    \frac{n*(n+1)}{2} - (n - 1) - 1= \frac{n*(n-1)}{2}
    $$
     The subscript of a\_nj is

    $$
    \frac{n*(n-1)}{2} + j - 1
    $$
     The subscript of a\_ij is

    $$
    \frac{i*(i-1)}{2} + j - 1
    $$

## 4. generalized tables

## 4.1 Definition of generalized tables

- A generalized table is a generalization of a linear table, also known as a list.
- It is generally written as LS = (a\_1, a\_2, --- , a\_n)

  - LS is the name of the generalized table
  - n is its length
- 3 Important Conclusions

  - Elements of a generalized table can be sub-tables, and elements of sub-tables can also be sub-tables

    ```
    A = (a, (b, c));
    ```
  - Generalized tables can be shared by other generalized tables

    ```
    A = ();
    B = (e);
    C = (a, (b, c, d));

    D = (A, B, C);
    // n = 3
    ```

    ! [Graphical representation of a generalized table](https://i-blog.csdnimg.cn/blog_migrate/3168d49cb5c2f7a2096d28b12eb603f4.png#pic_center)
  - A generalized table can be a recursive table

    ```
    E = (a, E);
    // n = 2
    ```

    ! [Graphical representation of a recursive generalized table](https://i-blog.csdnimg.cn/blog_migrate/5e2218b17c42f836cec706c101fa0a19.png#pic_center)
- 2 important operations

  - GetHead(LS)
    - The head of the fetched table is the first element of a non-empty generalized table.
      - can be a single atom
      - can be a single atom or a sub-table
  - GetTail(LS)
    - Fetch the tail of the table consisting of all elements except the head.

  ```
  AB = (a, b);
  C = (c);
  ABC = (AB, C);

  GetHead(ABC) == AB;
  GetTail(ABC) == (C);

  GetHead(AB) == a;
  GetTail(AB) == (b);

  GetHead(C) == c;
  GetTail(C) == ();
  ```

### 4.2 Generalized Table Storage Structure

- The data elements in a generalized table can have different structures (atoms, lists), so a chained storage structure is used

#### 4.2.1 Storage structure of a head-to-tail linked table

- Composition
  - Table node: represents the generalized table
    - Flag field
    - Flag field Pointer to the head of the table
    - Pointer to the end of the table
  - Atomic node: represents an atom
    - Flag field
    - Value field

```c
typedef enum {ATOM, LIST} ElemTag;				// 枚举
// ATOM == 0 : 原子
// ATOM == 1 : 子表

typedef struct GLNode {
    ElemTag tag;								// 用于区分原子结点和表结点
    
    union { 									// 联合
        AtomType atom;							// 原子结点值域，用户自行定义
        struct {								// 表结点的指针域
            struct GLNode *hp;					// 表头
            struct GLNode *tp;					// 表尾
        } ptr;
    };
} *Glist;
```

! [Storage structure for head-to-tail chained table representation](https://i-blog.csdnimg.cn/blog_migrate/f9911067284e5092cfacc8c253c419cd.png#pic_center)

- 3 Features
  - The head pointer to any non-empty generalized table points to a table node, except for an empty table which gets a null head pointer.
    - the hp field in that node points to the generalized table header (atomic node, table node)
    - The tp field points to the end of the generalized table (if the end of the table is empty, the pointer is empty, otherwise it is a table node)
  - Easily distinguish between atoms and sub-tables in the list at the hierarchical level
  - The number of table nodes at the highest level is the length of the generalized table.

#### 4.2.2 Extended Linear Chain Table Storage Structure

- Both atom and table nodes consist of 3 fields
  - Table node:
    - Flag field
    - Pointer to the head of the table
    - Pointer to the end of the table
  - Atomic node:
    - Flag field
    - Value field
    - Pointer field to the end of the table
