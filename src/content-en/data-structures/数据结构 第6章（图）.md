## 1. Definition and basic terminology of diagrams

### 1.1 Definition of a graph

- \*\*Graph\*\*\*G consists of two sets V and E, denoted G = (V , E )

  - V : an exhaustive non-empty set of vertices.
    - V(G): the set of vertices of the graph G
  - E : the set of even pairs of vertices in V .
    - E(G): the set of edges of the graph G.
      - E(G) can be empty: the graph G has only vertices and no edges.
      - E(G) is the set of directed edges: a directed graph
      - E(G) is the set of undirected edges: an undirected graph.
- In a directed graph, vertex pair <x, y> is ordered

  - <x, y>: a directed edge from vertex x to vertex y

    - x is the start point of the directed edge, y is the end point of the directed edge.
    - <x, y> is also called an arc, with x being the end of the arc and y being the beginning.

  ! [Example of a directed graph](https://i-blog.csdnimg.cn/blog_migrate/74272b608a688367152dc5ac17715310.png#pic_center)
- In an undirected graph, the vertex pair (x, y) is unordered

  - (x, y): an edge associated with a vertex x and a vertex y.

  ! [Example of an undirected graph](https://i-blog.csdnimg.cn/blog_migrate/29bba13094c6d46364328a2ded2b195c.png#pic_center)

### 1.2 Basic graph terminology

- Denote by n the number of vertices in a graph, and by e the number of edges
- **Subgraph**: Suppose there are 2 graphs G = (V, E) and G' = (V', E'), V' ⊆ V and E' ⊆ E, G' is a subgraph of G

  ! [Examples of subgraphs](https://i-blog.csdnimg.cn/blog_migrate/497e5cd1114e50874b0f2a497b010f2b.png#pic_center)
- **Undirected complete graph**: for an undirected graph, has n(n-1)/2 edges
- **Directed complete graph**: for a directed graph, has n(n-1) arcs.
- **Sparse graph**: graph with few edges or arcs (e.g. e < nlog\_2(n))
- **dense graph**: graph with more edges or arcs
- **Weight**: a value with some meaning on each edge
- **net**: a graph with weights
- **neighborhood**: for an undirected graph G with edges (v, v') ∈ E, v and v' are **neighborhoods** of each other, and v and v' are **adjacent** to each other

  - Edge (v, v')** is attached to ** vertices v and v'
  - Edge (v, v') is **associated** with vertices v and v' **adjacent
- **degree**: the number of edges associated with vertex v, denoted TD(v)
- **in-degree**: the number of arcs starting at vertex v, denoted ID(v)
- **outdegree**: the number of arcs ending at vertex v, denoted OD(v)
- **Path**: from vertex v to vertex v'

  - Undirected graph: sequence of vertices (v = v\_ { i , 0 }, v\_ { i , 1 }, --- , v\_ { i , 0 } = v'), (v\_ { i , j-1 } , v\_ { i , j }) ∈ E , 1 ≤ j ≤ m
  - Directed graph: paths are also directed and the sequence of vertices should satisfy <v\_ { i , j-1 } , v\_ { i , j }> ∈ E , 1 ≤ j ≤ m
- **Path length**: the number of edges or arcs that a path passes through
- **loop (ring)**: a path where the first di-vertex and the last vertex are the same
- **simple path**: paths where vertices do not recur in the sequence
- **Simple loop (simple ring)**: a loop in which the vertices do not recur except for the first and last vertex
- **connected**: in an undirected graph G, there is a path from vertex v to vertex v', and v and v' are **connected**
- **connected graph**: for any two vertices v\_i , v\_j ∈ V in the graph, v\_i and v\_j are connected
- **connected component**: extremely connected subgraph in an undirected graph
- **strongly connected graph**: in a directed graph G, for every pair v\_i , v\_j ∈ V , v\_i ≠ v\_j , there exists a path from v\_i to v\_j and from v\_j to v\_i
- **strongly connected component**: extremely strongly connected subgraph in a directed graph
- **connected graph spanning tree**: a minimal connected subgraph that contains all the vertices in the graph but only n-1 edges sufficient to form a tree
- **directed tree**: a directed graph with one vertex of incidence 0 and all other vertices of incidence 1
- **Generative forest**: a directed graph consisting of several directed trees containing all the vertices in the graph, but only enough arcs to form several disjoint directed trees

## 2. The storage structure of a graph

## 2.1 The adjacency matrix

#### 2.1.1 Adjacency matrix representation

- Adjacency Matrix**: A matrix representing the adjacency between vertices.
- G (V , E) is a graph with n vertices, and the adjacency matrix of G is a square matrix of order n with the following properties

  $$
  A[i][j]=\begin{cases} 1 \quad 若 <v_i, v_j> 或 (v_i, v_j) ∈ E\\ 0 \quad 反之\end{cases}
  $$.
- G is a net, then its adjacency matrix is

  $$
  A[i][j]=\begin{cases} w_{i,j} \quad 若 <v_i, v_j> 或 (v_i, v_j) ∈ E\\ ∞ \quad 反之\end{cases}
  $$
- The adjacency matrix storage representation of the graph

```c
#define MAXSIZE 10

typedef struct {

    int arcnum;										// 图的当前边数
```

    }

    printf("Create Success\n");
}

    int i = 0;
    for (; i < G->vexnum; i++) {
        if (G->vexs[i] == vex) {
            break;
        }
    }
    return i;
}

#### 2.1.3 Advantages and Disadvantages of Adjacency Matrix Representation

- Advantages

  - Easy to determine whether there is an edge between two vertices according to A[ i ] [ j ] = 0 or 1.
  - Easy to calculate the degree of each vertex
    - Undirected graph: the sum of the elements of row i of the adjacency matrix is the degree of vertex i.
    - Undirected graph: the sum of the elements of row i of the adjacency matrix is the degree of vertex i. Directed graph: the sum of the elements of row i is the out degree of vertex i, the sum of the elements of column i is the in degree of vertex i.
- Disadvantages

  - It is not easy to add or remove vertices.
  - Not easy to count the number of edges, need to scan all the elements of the critical matrix to finish counting, time complexity is O(n^2)
  - High space complexity
    - Directed graph: n vertices require n^2 cells to store edges.
    - Undirected graph: n vertices require n(n-1)/2 units for compressed storage method.
    - Undirected graphs: n vertices require n(n-1)/2 units for compressed storage.

### 2.2 Adjacency Tables

#### 2.2.1 Adjacency Table Representation

- Adjacency List**: a chained storage structure for graphs.
  - Create a single linked table for each vertex v\_i in the graph, and put the vertices adjacent to v\_i in this linked table.
  - The first node of each list holds information about the vertex, think of this node as the head of the list, and the rest of the nodes hold information about the edges
- Composition
  - ** table header node table **: stored by the table header node in the form of a sequential structure
    - **data**: stores the name of vertex v\_i or other related information.
    - **chain field (firstarc)**: points to the first node in the chain table (the first neighbor adjacent to vertex v\_i)
  - **Edge table**: consists of a list of 2n edge links representing the relationships between vertices in the graph.
    - **adjacent point domain (adjvex)**: the position in the graph of the point adjacent to vertex v\_i
    - **Data field (info)**: store the information related to the edges, such as weights etc.
    - **chain domain (nextarc)**: the node of the next edge adjacent to vertex v\_i

```c
#define  MAXSIZE 10									// 最大顶点数

typedef struct ArcNode {							// 边结点
    int adjvex;                         			// 该边所指向的顶点的位置
    struct ArcNode* nextarc;            			// 指向下一条边的指针
    char info;                          			// 和边相关的信息
} ArcNode;

typedef struct VNode {								// 顶点信息
    int data;
    ArcNode* firstarc;                  			// 指向第一条依附该顶点的边的指针
} VNode, AdjList[MAXSIZE];               			// AdjList 表示邻接表类型

typedef struct {									// 邻接表
    AdjList vertices;
    int vexnum, arcnum;                 			// 图的当前顶点数和边数
} ALGraph;
```

```c
void CreateUDG(ALGraph* G) {
    printf("Enter vexnum , arcnum: ");
    scanf("%d,%d", &(G->vexnum), &(G->arcnum));		// 输入总定点数，总边数

    for (int i = 0; i < G->vexnum; i++)				// 输入各点，构造表头结点表
    {
        printf("Enter vex: ");
        scanf("%d", &(G->vertices[i].data));		// 输入顶点值
        G->vertices[i].firstarc = NULL;				// 初始化表头结点的指针域为 NULL
    }

    for (int i = 0; i < G->arcnum; i++)				// 输入各边，构造邻接表
    {
        int vex1, vex2;
        printf("Enter vex1, vex2: ");

        int i = LocateVex(G, vex1);					// 获取 vex1 在 G 中的序号（从 0 开始）
        int j = LocateVex(G, vex2);					// 获取 vex2 在 G 中的序号（从 0 开始）

        ArcNode* p1 = (ArcNode*)malloc(sizeof(ArcNode)); // 生成新的边结点 p1
        p1->adjvex = j;								// 邻接点序号为 j
        p1->nextarc = G->vertices[i].firstarc;		
        G->vertices[i].firstarc = p1;				// 将新结点 p1 插入顶点 v_i 的边表头部

        ArcNode* p2 = (ArcNode*)malloc(sizeof(ArcNode)); // 生成新的边结点 p2
        p2->adjvex = i;								// 邻接点序号为 i
        p2->nextarc = G->vertices[j].firstarc;
        G->vertices[j].firstarc = p2;				// 将新结点 p2 插入顶点 v_j 的边表头部
    }

    printf("Create Success\n");
}

int LocateVex(ALGraph* G, int vex) {
    for (int i = 0; i < G->vexnum; i++) {
        if (G->vertices[i].data == vex)
            return i;
    }
}
```

#### 2.2.3 Advantages and disadvantages of the adjacency table representation

- Advantages

  - Easy to add and remove vertices
  - Easy to count the number of edges, scanning all the edge tables in the order of the vertex table can get the number of edges, with a time complexity of O(n+e).
  - Time complexity is O(n+e)
    - Undirected graph: n vertices and e edges, n vertex table nodes and 2e edge table nodes in the adjacency table.
    - Directed graph: n-vertex, e-edge graph with n vertex table nodes and e edge table nodes in the adjacency table.
    - The space complexity is O(n+e).
- Disadvantages

  - It is not easy to determine whether there is an edge between vertices. To determine whether there is an edge between v\_i and v\_j, we need to scan the ith edge table, which takes O(n) time in the worst case.
  - It is not easy to calculate the degree of each vertex of a directed graph.
    - Undirected graph: the degree of vertex v\_i is the number of nodes in the ith edge table.

## 3. Graph Iteration

## 3.1 Depth-first search

#### 3.1.1 Depth-first search traversal process

- **Depth First Search (DFS)**: a generalization of prior order traversal of a tree.
- Procedure
  - Starting from a vertex v in the graph, visit v
  - Find the just visited vertex and the first unvisited neighbor, visit that vertex; use that vertex as the new vertex, repeat until the just visited vertex has no unvisited neighbors
  - Return to the previous visited vertex that still has unvisited neighbors, find the next unvisited neighbor of that vertex, and visit that vertex.
  - Repeat steps 2 and 3 until all vertices in the graph have been visited.

! [Depth-first traversal process](https://i-blog.csdnimg.cn/blog_migrate/da6527620321986e82d8d24108cb780d.png#pic_center)

- Depending on the paths and nodes at the time of the visit one can form a **depth-first spanning tree**

### 3.2 Breadth-first search

#### 3.2.1 The process of breadth-first search traversal

- Breadth First Search (BFS)**: similar to hierarchical tree traversal.
- Process
  - Starting from a vertex v in the graph, visit v
  - Visit each unvisited neighbor of v in turn
  - from these neighbors in turn, so that the "neighbors of the first-visited vertex" are visited before the "neighbors of the later-visited vertex"; repeat this step until all the neighbors of the visited vertices in the graph are visited. visited

## 4.1 Minimal Spanning Trees

- **Minimum Cost Spanning Tree (Minimum Spanning Tree)**: the spanning tree that minimizes the sum of the costs of the edges among all spanning trees of a connected net

#### 4.1.1 Pulliam's Algorithm

- Process.

  Assume that N = { V , E } is a connected network and TE is the set of edges in the minimum spanning tree on N

  - U = { u\_0 }(u\_0 ∈ V), TE = {}
  - Find an edge (u\_0 , v\_0 ) with minimum weight among all edges (u , v ) ∈ E of u ∈ U , v ∈ V-U and merge it into the set TE , while v\_0 is merged into U .
  - Repeat the second step until U = V

#### 4.1.2 Kruskal's Algorithm

- Procedure

  Assuming that N = { V , E } is a connected network, arrange the edges in N in order of weights from smallest to largest

  - The initial state is an unconnected graph T = (V, {}) with n vertices and no edges, where each vertex in the graph is a connected component.
  - Select the edge with the lowest weight in E. If the vertex attached to this edge falls on a different connected component of T (without forming a loop), then add this edge to T. Otherwise, discard this edge and select the next edge with the lowest weight.
  - Repeat the second step until all the vertices in T are on the same connectivity component

### 4.3 Topological ordering

#### 4.3.1 AOV-nets

- **Directed Acycline Graph (DAG)**: a directed graph without loops
- **Activity On Vertex Network (AOV-net)**: directed graph with vertices representing activities and arcs representing priority relations between activities
- **Topological ordering**: arranges all vertices in an AOV-net into a linear sequence.
  - The sequence satisfies: if there is a path from vertex v\_i to vertex v\_j in the AOV-net, then vertex v\_i must precede vertex v\_j in the linear sequence.

#### 4.3.2 Process of topological ordering

- Select a vertex without predecessors in the directed graph and output it.
- Remove the vertex and all arcs ending with it from the graph.
- Repeat steps 1 and 2 until there are no more vertices without predecessors.
- If the number of vertices output is less than the number of vertices in the directed graph, then there is a ring in the directed graph, otherwise the sequence of vertices output is a topological ordering

### 4.4 Critical paths

#### 4.4.1 AOE-nets

- **Activity On Edge Network (AOV-net)**: represents activities by edges, directed acyclic graph with weights

  - Vertex: events
  - Arc: activity
  - Weights: events for which the activity lasts
- Vertex: event Arc: activity Right: event for which the activity lasts **Source**: only one point in the net with an incidence of 0
- **Sink**: only one point in the net that has an out-degree of 0
- **Path length with weights (path length)**: sum of the weights on each arc of a path
- **Critical Path**: a path with the longest weighted path length from the source to the sink.
- **Critical Activity**: activity on a critical path

#### 4.4.2 Critical Path Sequence Solving Procedure

- Order the vertices in the graph, and find the earliest occurrence time of each event ve(i) by topological ordering during the ordering process
- Find the latest occurrence time of each event by topological ordering vl(i)
- Find the earliest start time of each event a\_i e(i)
- Find the latest start time of each event a\_i l(i)
- Find the activity a\_i for which e(i) = l(i), i.e., the critical activity; each path from the source to the sink formed by the critical activity is a critical path, and there may be more than one critical path
