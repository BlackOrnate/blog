## 1. Definition and characteristics of stacks and queues

### 1.1 Definition and characteristics of stacks

- **stack**: a linear table limited to insertion or deletion operations only at the end of the table
  - top: the end of the table.
  - ** bottom (bottom) **: the head end of the table
  - **Empty stack**: empty table without elements
  - **Last In First Out (LIFO)** linear table: the first element to exit the stack is the top element.

### 1.2 Definition and characteristics of a queue

- **queue**: **First In First Out (FIFO)** linear table
  - Only allow insertion at one end of the table, while the other end of the deletion of elements
  - **rear**: the end of the table where insertions are allowed
  - **Front**: the end where deletions are permitted

## 2. Representation of stacks and implementation of operations

## 2.1 Representation and implementation of sequential stacks

- A stack implemented using a sequential storage structure
- A sequential set of memory cells holds the data elements from the bottom of the stack to the top of the stack, with a pointer at the top indicating the position of the top element in the sequential stack.

### 2.2 Chained Stack Representation and Implementation

- Stacks implemented using a chained storage structure
- Usually a chained stack is represented as a single linked table

```c
typedef struct StackNode {
    ElemType data;
    struct StackNode *next;
} StackNode, *LinkStack;
```

! [Diagram of a chain stack](https://i-blog.csdnimg.cn/blog_migrate/dfc34e02e63b3618c16a9e0018c77a12.png#pic_center)

- Since the main role of the stack is to insert and delete at the top of the stack, it is convenient to use the head of the chained table as the top of the stack, and there is no need to attach a head node for ease of operation, as in the case of a single-chained table

#### 2.2.1 Initialization

```c
void InitStack(LinkStack* S) {
	*S = (LinkStack)malloc(sizeof(StackNode));
	(*S)->stacksize = 0;
	(*S)->next =  NULL;
	printf("Init Success\n");
}
```

#### 2.2.2 Stacking

- Steps
  - Allocate space for the incoming element e, pointed to by pointer p.
  - Set the data field of the new node to e
  - Insert the new node into the top of the stack
  - Modify the top-of-stack pointer to p

```c
void Push(LinkStack* S, ABC abc) {
	LinkStack new = (LinkStack)malloc(sizeof(StackNode));
    
	new->abc = abc;
    
	new->next = (*S)->next;
    
	(*S)->next = new;
    
	(*S)->stacksize++;
	printf("Push Success\n");
}
```

#### 2.2.3 Out of the stack

- Steps
  - Assign the top element to e
  - Temporarily save the space of the top stack element for freeing.
  - Modify the top-of-stack pointer to point to the new top-of-stack element.
  - Free the space of the original top-of-stack element

```c
void Pop(LinkStack* S) {
	StackNode* node = (*S)->next;
    
	(*S)->next = node->next;
    
	(*S)->stacksize--;
    
	free(node);
    
	printf("Pop Success\n");
}
```

## 3.1 Problems Solved with Recursive Algorithms

- Recursion: the application of a definition of a function, procedure, or data structure that occurs directly (or indirectly) inside the definition of the function, procedure, or data structure itself.

#### 3.1.1 Definitions are recursive

- The factorial function

  $$
  Fact(n)=\begin{cases} 1 \quad\quad\quad\quad\quad\quad\quad\quad n=0 \\ n*Fact(n-1)\quad\quad n>0\end{cases}
  $$

  ```python
  int Fact(int i) {
      if (i == 0) {
          return 0;
      } else {
          return i*Fact(i-1);
      }
  }
  ```
- Second-order Fibonacci series

  $$
  Fib(n)=\begin{cases} 1 \quad\quad\quad\quad\quad\quad\quad\quad\quad\quad\quad n=1或n=2 \\ Fib(n-1)+Fib(n-2)\quad\quad 其他情形\end{cases}
  $$

  ```python
  int Fib(int i) {
      if (i == 1 || i == 2) {
          return 1;
      } else {
          return Fib(n-1) + Fib(n-2);
      }
  }
  ``` Second-order Fibonacci series
- Recursive solving: solving a complex problem by decomposing it into several relatively simple subproblems with the same or similar solutions
- Partitioning: decomposition-solution strategy

  - Conditions of use
    - Can transform a problem into a new problem, and the new problem and the original problem solution is the same or similar, the only difference is the object to deal with, and these objects to deal with smaller and change regularly
    - The problem can be simplified by the above transformations
    - There must be a clear recursive exit, or recursive boundary.

#### 3.1.3 The solution to the problem is recursive

- Hanoi Tower Problem, Eight Queens Problem, Maze Problem, etc.
- Recursive algorithm for the Hanoi Tower problem

  - Steps
    - If n=1, move disks numbered from 1 to n-1 on A to B and C as an auxiliary tower.
    - Move the disk numbered n directly from A to C.
    - Recursively, move disks numbered from 1 to n-1 on B to C, with A as the auxiliary tower.

  ```c
  #define count 0;

  void Hanoi(int n, char A, char B, char C) {
      if (n == 1) {
          move(A, 1, C);
      } else {
          Hanoi(n-1, A, C, B);
          move(A, n, C);
          Hanoi(n-1, B, A, C);
      }
  }

  void move(char A, int n, char C) {
      printf("%d, %d, %c, %c", ++count, n, A, C);
  }
  ```

### 3.2 Recursive Procedures and Recursive Workstacks

- Linking and exchanging information between the calling and called functions takes place via the stack.
- When a function is called during the runtime of another function

  - When a function is called during the runtime of another function, the system does three things before running the called function.
    - Pass all the real parameters, return address and other information to the called function for storage.
    - Allocate storage for the called local variables.
    - Transfer control to the entry point of the called function
  - Before returning from the called function to the calling function, the system should also accomplish 3 things
    - Save the results of the called function's calculations
    - Release the data area of the called function
    - Transfer control to the calling function according to the return address saved by the called function.
- When more than one function constitutes a nested call, according to the principle of "call first, return first", the information transfer between the above functions and control transfer must be realized through the "stack".

  - The system arranges the data space required for the operation of the entire system in a stack
  - Whenever a function is called, it is allocated a storage area on the top of the stack.
  - Whenever a function is called, it is allocated a memory area on the top of the stack.
  - Whenever a function is called, it is allocated an area of storage on the top of the stack.
- In order to ensure that the recursive function is executed correctly, the system needs to set up a "recursive workstack" as the data storage area for the entire duration of the recursive function.

  - Each level of recursion requires information to form a work record, including all real parameters, all local variables, as well as the return address of the previous level.
  - For each level of recursion, a new working record is generated and pressed onto the top of the stack.
  - Every time you exit a level of recursion, a new workbook is popped off the top of the stack.
  - The work record at the current execution level must be the work record at the top of the recursive work stack, which is called the "**Activity Record**"

### 3.4 Converting Recursion to Non-Recursion Using a Stack

- Steps
  - Set up a working stack to chafe the recursive working record (including real parameters, return addresses, local variables, etc.).
  - Enter the non-recursive call entry (at the beginning of the called program) and put the actual parameters and return address from the calling program on the stack.
  - Entering the recursive call entry
    - When the end condition of recursion is not met, recursion is made layer by layer, and the actual parameter, return address and local variables are put on the stack (this process can be realized by using a loop statement).
    - When the recursive end condition is met, the constant that reaches the exit of the recursion is used as the current function value.
  - Return processing
    - In the case that the stack is not empty, repeatedly exit the top of the stack record, according to the return address in the record to perform the operation specified in the question
      - Calculate the current function value layer by layer until the stack is empty.

## 4.1 Circular Queues - Sequential Representation and Implementation of Queues

- In the sequential storage structure of a queue, in addition to storing the elements from the head of the queue to the tail of the queue in a set of consecutively-addressed memory cells, two integer variables, front and rear, are required to indicate the positions of the head and tail elements, respectively.

```c

typedef struct {
    ElemType *elem;							// 存储空间的基地址
    int front;								// 头指针
    int rear;								// 尾指针
} SqQueue;
```

- When initializing an empty queue, front=rear=0.

  ! [Queue entry for sequential queues](https://i-blog.csdnimg.cn/blog_migrate/337e77e47fdb51eff7404c7f05019783.png#pic_center)

	Q->abc = (ABC*)malloc(MAXSIZE * sizeof(ABC));
	Q->front = Q->rear = 0;
	printf("Init Success\n");

	printf("length = %d\n", (Q.rear - Q.front + MAXSIZE) % MAXSIZE);
}

void EnQueue(SqQueue* Q, ABC abc) {
	if ((Q->rear + 1) % MAXSIZE == Q->front)
	{
		printf("En Error\n");
		return 0;
	}

	Q->abc[Q->rear] = abc;
	Q->rear = (Q->rear + 1) % MAXSIZE;
	printf("En Success\n");
}

	if (Q->front == Q->rear)
	{
		printf("De Error\n");
		return 0;
	}

	Q->front = (Q->front + 1) % MAXSIZE;
	printf("De Success\n");
}

	if (Q.front != Q.rear)
	{
		ABC abc = Q.abc[Q.front];

	}

### 4.2 Chained Queues - Chained Representation and Implementation of Queues

- A queue implemented using a chained storage structure
- Usually, a chained queue is represented as a single linked table

```c
typedef struct QNode {
    ElemType data;
    struct QNode *next;
} QNode, *QueuePtr;

typedef struct {
    QueuePtr front;							// 队头指针
    QueuePtr rear;							// 队尾指针
} LinkQueue;
```

#### 4.2.1 Initialization

- Steps
  - Generate a new node as the head node, with pointers to the head and tail of the queue pointing to this node.
  - The pointer field of the head node is left empty.

The head node pointer field is left empty.

#### 4.2.2 Entering the queue

- Procedure
  - Allocate node space for the incoming element, pointing to it with pointer p.
  - Set the data field of the new node to e
  - Insert the new node at the end of the queue
  - Change the pointer to the end of the queue to p

```c
void EnQueue(LinkQueue* Q, ABC abc) {
	QNode* node = (QNode*)malloc(sizeof(QNode));

	node->abc = abc;
	node->next = NULL;
	Q->rear->next = node;
	Q->rear = node;
	printf("En Success\n");
}
```

#### 4.2.3 Out of queue

- Steps
  - Temporarily save the space of the head element of the queue to be released.
  - Modify the pointer field of the head node to point to the next node.
  - Determine whether the outgoing element is the last element
    - Determine if the queue element is the last element Yes: reassign the pointer to the end of the queue to point to the head node
  - Free the space of the original head element.

```c
void DeQueue(LinkQueue* Q) {
	if (Q->front == Q->rear)
	{
		printf("De Error\n");
		return 0;
	}

	QNode* node = Q->front->next;
	Q->front->next = node->next;
	free(node);
	printf("De Success\n");
}
```
