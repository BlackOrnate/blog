## 1. Definitions of trees and binary trees

## 1.1 Definition of a tree

- **Tree**: finite set of n (n ≥ 0) nodes
  - Empty tree: n = 0
  - Non-empty tree T :
    - There is one and only one node called the root
    - The nodes other than the root can be divided into m (m > 0) disjoint finite sets T\_1 , T\_2 , - - - , T\_m
      - Each of these sets is itself a tree, called the root's subtree (SubTree)

! [Example of a tree](https://i-blog.csdnimg.cn/blog_migrate/914071cfdca8cd8b742040e7b41d77f5.png#pic_center)

### 1.2 Basic Tree Terminology

- **Node**: an independent unit of the tree species, containing a data element and several branches pointing to its subtrees
- **degree of node**: the number of subtrees a node has
- **Tree degree**: the maximum value of the degree of each node in the tree
- **Leaf (terminal node)**: node with degree 0
- **Non-terminal nodes (branch nodes)**: nodes whose degree is not 0; except for the root node, non-terminal nodes are also called internal nodes
- **Parents and children**:
  - The root of a subtree of a node is called the child of that node
  - The node is called the child's parent
- **Brothers**: children of the same parent are called brothers to each other
- **Ancestors**: all the nodes on the branches from the root to the node
- **Children**: any node of a subtree species with a node as root
- **Hierarchy**: defined from the root, with the root as the first level, the children of the root as the second level, and so on
- **Cousins**: nodes with both parents at the same level are cousins to each other
- **Tree depth (height)**: the maximum level of a node in a tree
- **Ordered and unordered trees**:
  - Ordered tree: each child of a node in the tree is ordered from left to right (not interchangeable)
    - First child: the root of the leftmost subtree
    - Last child: the root of the rightmost subtree.
  - Unordered tree: the opposite of an ordered tree
- **Forest**: the set of m (m ≥ 0) trees that do not intersect each other.

### 1.3 Definition of a binary tree

- **Binary Tree**: a set of n (n ≥ 0) nodes.

  - Empty tree: n = 0
  - Non-empty tree T :
    - There is one and only one node called the root.
    - The nodes other than the root are divided into two disjoint subsets, T\_1 and T\_2, called the left and right subtrees of T respectively.
      - T\_1 and T\_2 are themselves both binary trees.
- The difference between a binary tree and a tree

  - A binary tree has at most two subtrees per node.
    - There is no node of degree greater than 2 in a binary tree.
  - A binary tree has left and right subtrees, the order of which cannot be reversed arbitrarily.
- The 5 basic forms of a binary tree

  - Empty binary tree
  - A binary tree with only a root node
  - A binary tree with an empty right subtree
  - A binary tree whose left and right subtrees are not empty.
  - Binary tree with empty left subtree

## 3. Traversing binary trees and clue binary trees

## 3.1 Traversing binary trees

- Traversing a binary tree: Patrolling each node of the tree along a search path such that each node is visited once and only once.
- Traversing a binary tree in first order

  - Visiting the root node
  - Traverse the left subtree in the first order
  - Traverse right subtree in first order
- Traversing a binary tree in middle order

  - Traversing the left subtree in the middle order
  - Accessing the Root Node
  - Traversing the right subtree in the middle order
- Traversing a binary tree in the second order

  - Posterior traversal of the left subtree
  - Posterior traversal of the right subtree
  - Accessing the root node
- First order traversal: top to bottom, left to right
- Middle-order traversal: left-to-right, top-to-bottom
- Posterior traversal: bottom to top, left to right

#### 3.1.1 Recursive Algorithm for Middle Order Traversal

```c
void InOrderTraverse(BiTree T) {
	if (T) {
		InOrderTraverse(T->lchild);					// 中序遍历左子树
		printf("value = %d\n", T->value);			// 访问根节点
		InOrderTraverse(T->rchild);					// 中序遍历右子树
	}
}
```

#### 3.1.2 Non-recursive algorithm for mid-order traversal

- Steps
  - Initialize an empty stack S with pointer p to the root node.
  - Request a node space q to hold the popped elements on the top of the stack.
  - When p is not empty or the stack S is not empty, loop through the following operations.
    - When p is not empty or stack S is not empty, p goes into the stack and p points to the left child of the node.
    - When p is empty or stack S is not empty, p goes on the stack, and p points to the right child of the node.

```c
void InOrderTraverse(BiTree T) {
    BiTree* p = T;
    BiNode q = (BiNode*)malloc(sizeof(BiNode));
    
    InitStack(S);
    while (p || StackEmpty(S)) {
        if (p) {									// p 非空
            Push(S, p);								// 根指针进栈
            p = p->lchild;							// 遍历左子树
        } else {
            Pop(S, q);								// 退栈
            printf("value = %d\n", T->value);		// 访问根节点
            p = q->rchild;							// 遍历右子树
        }
    }
}
```

#### 3.1.5 Calculating the depth of a binary tree

```python
int Depth(BiTree T) {
	if (T)
	{
		int i = Depth(T->lchild);					// 递归计算左子树的深度记为 i
		int j = Depth(T->rchild);					// 递归计算右子树的深度记为 j
		return (i > j) ? i + 1 : j + 1;
	}
	else
	{
		return 0;									// 递归结束，深度为 0
	}
}
```

### 3.2 Clue Binary Trees

- **clue chain table**: a binary chain table with a special node structure as the storage structure of a binary tree
  - **Clue**: pointers to node predecessors and successors
  - **Thread Binary Tree**: binary tree with threads
  - **Threading**: the process of traversing a binary tree in some order to turn it into a thread binary tree
  - Node form:
    - Left child
    - Left flag
      - 0 : points to the left child of the node
      - 1 : points to the predecessor of the node
    - Data
    - Right flag
      - 0 : points to the right child of the node
      - 1: points to the successor of the node
    - Right child

_```c
typedef struct BiThrNode {
    ElemType elem;
    struct BiThrNode *lchild, *rchild;				// 左右孩子指针
    int LTag, RTag;									// 左右标志
} BiThrNode, *BiThrTree;
```.

#### 3.2.1 Clueing in the middle order of a subtree rooted at node p

- Steps
  - p is not null, left subtree recursive cluing
  - p's left child is null, add a left clue to p, set LTag to 1, and let p's left child pointer point to pre.
    - non-empty, set LTag of p to 0
  - pre's right child is empty, add right clue to pre, set RTag to 1, let pre's left child pointer point to p (successor)
    - non-null, set RTag of pre to 0
  - Point pre to the node it just visited
    - pre = p
  - Recursive serialization of the right subtree

```
void InThreading(BiThrTree p) {
    if (p) {
        InThreading(p->lchild);						// 左子树递归线索化

        if (!p->lchild) {							// p 的左孩子为空
            p->LTag = 1;							// 给 p 加上左线索
            p->lchild = pre;						// 让 p 的左孩子指针指向 pre （前驱）
        }
        else {
            p->LTag = 0;
        }

        if (!pre->rchild) {							// pre 的右孩子为空
            pre->RTag = 1;							// 给 pre 加上右线索
            pre->rchild = p;						// 让 pre 的左孩子指针指向 p （后继）
        }
        else {
            pre->RTag = 0;
        }

        pre = p;									// 保持 pre 指向 p 的前驱

        InThreading(p->rchild);						// 右子树递归序列化
    }
}
```

#### 3.2.3 Traversing a clued binary tree

- Find in the middle-order cue binary tree

  - Find the predecessor of the node pointed to by the p-pointer.
    - p->LTag = 1: the left chain of p indicates its predecessor.
    - p->LTag = 0: p has a left subtree, and the predecessor of the node is the last node visited when traversing the left subtree (the right-most down node in the left subtree).
  - Find the successor of the node pointed to by the p pointer
    - p->RTag = 1: p's right chain indicates its successor.
    - p->RTag = 0: p has a right subtree, and the node's successor is the first node visited when traversing the right subtree (the left-most down node in the right subtree)
- Finding in a prior-order trail binary tree

  - Finds the predecessor of the node pointed to by the p pointer
    - p->LTag = 1: p's left chain indicates its predecessor.
    - p->LTag = 0: p has left subtree
      - Predecessor of node
        - \*p is the left child of its parent, then its predecessor is its parent.
        - \*p is the right child of its parent, then its predecessor is the two-point node last visited in a prior traversal of its parent's left subtree.
  - Find the successor of the node pointed to by the p pointer
    - p->RTag = 1: p's right chain indicates its successor
    - p->RTag = 0: p has a right subtree, \*p's successor is the root of its left subtree (if it exists) or the root of its right subtree.
- Finding in a back-ordered trail binary tree

  - Finds the predecessor of the node pointed to by p's pointer

    - p->LTag = 1: p's left chain indicates its predecessor.
    - p->LTag = 0:
      - p->RTag = 1: p's left chain indicates its predecessor.
      - p->RTag = 0: p's right chain indicates its predecessor.
  - Find the successor of the node pointed to by the p pointer.

    - \*p is the root of a binary tree: the successor is empty.
    - \*p is the right child of its parent: the successor is the parent node.
    - \*p is the left child of its biparent:

      - \*p has no right brother: the successor is a biparental node.
      - \*p has a right brother: successor is the first node traversed in posterior order in the right subtree of the biparent (the left-most down leaf node in the right subtree)

##### Traversing a middle-order trail binary tree

- Steps
  - Pointer p points to the root node
  - If p is a non-empty tree or if the traversal is not complete, the loop does the following
    - Go down the left child to the bottom left node, \*p, which is the first node in the middle order.
    - Accessing \*p
    - Iterate along the right trail to find the successor node to the current node, \*p, and visit the successor node until the right trail reaches 0 or the traversal is complete.
    - Go to the right subtree of p

```c
void InOrderThraverse_Thr(BiThrTree T)
{
    pre = T->lchild;								// p 指向根结点

    while (pre != T)								// 空树或遍历结束时，pre == T
    {
        while (pre->LTag == 0)
        {
            pre = pre->lchild;						// 沿左孩子向下
        }

        printf("value = %d\n", pre->value);			// 访问其左子树为空的结点

        while (pre->RTag == 1 && pre->rchild != T)
        {
            pre = pre->rchild;						// 沿右线索
            printf("value = %d\n", pre->value);		// 访问后继结点
        }

        pre = pre->rchild;							// 转向 p 的右子树
    }
}
```

### 4.2 Conversion of forests to binary trees

#### 4.2.1 Forest to binary tree conversion

- F = { T\_1 , T\_2 , - - - , T\_m } for a forest, converted to a binary tree B = { root , LB , RB }
  - F is empty (m = 0): B is an empty tree
  - F is non-empty (m ≠ 0):
    - root of B: root of the first tree in the forest ROOT( T\_1 )
    - Left subtree of B LB: binary tree converted from the forest of subtrees of the root node in T\_1 F\_1 = { T\_11 , T\_12 , ---- , T\_1m }
    - Right subtree of B RB : Binary tree converted from forest F' = { T\_2 , T\_3 , ---- , T\_m }

#### 4.2.2 Conversion of a binary tree to a forest

- B = { root , LB , RB } is a binary tree converted to a forest F = { T\_1 , T\_2 , ---- , T\_m }
  - B is empty: F is empty
  - B is non-empty:
    - ROOT( T\_1 ) of the first tree T\_1 in F: root root of B
    - Forest of subtrees of the root node in T\_1 F\_1: Forest converted from LB, the left subtree of B.
    - Forest of trees other than T\_1 F' = { T\_2 , T\_3 , ---- , T\_m }: forest converted from RB, the right subtree of B.

### 4.3 Traversal of Trees and Forests

#### 4.3.1 Tree Traversal

- Root-first traversal: first visit the root node of the tree, then traverse each subtree first.
- Posterior root traversal: first traverse each subtree sequentially posterior to the root, then visit the root node.

#### 4.3.2 Forest Traversal

- Traverse the forest in first order

  - Accessing the root node of the first tree in the forest
  - Traverse the forest of subtrees of the root node of the first tree in first order.
  - Traverse the forest of the remaining trees after the first tree in first order
- Traverse the forest in middle order

  - Iterate through the forest of subtrees at the root node of the first tree in intermediate order.
  - Accessing the root node of the first tree in the forest
  - Traverse the forest of the remaining trees after removing the first tree in middle order

## 5. Huffman trees and their applications

## 5.1 Basic concepts of Huffman trees

#### 5.1.1 Definition of a Huffman tree

- **Huffman Tree (Optimal Tree)**: a class of trees with the shortest weighted path lengths

#### 5.1.2 Basic Terminology of Huffman Tree

- **Path**: the branches from one node to another node in the tree form the path between these two nodes
- **Path length**: number of branches on the path
- **Path length of the tree**: the sum of the path lengths from the root of the tree to each node
- **right**: numerical description of an entity

  - Node weights
  - Edge weights
- **Path length with weights of a node**: the product of the length of the path from that node to the root of the tree and the full on the node
- **Weighted path length of the tree**: the sum of the weighted path lengths of all leaf nodes in the tree, denoted as

  $$
  WPL = \sum_{k=1}^{n}{w_kl_k}
  $$.
- **Huffman tree**: m weights { w\_1 , w\_2 , --- , w\_m }, a binary tree with n leaf nodes can be constructed, and the weight of each leaf node is w\_i , then the binary tree with the smallest weighted path length, WPL, is called an optimal binary tree or Huffman tree

### 5.2 Algorithm for constructing a Huffman tree

#### 5.2.1 Huffman tree construction process

- According to the given n weights { w\_1 , w\_2 , ---- , w\_n }, construct n binary trees with only root nodes, these n binary trees form a forest F.
- In the forest F, select the two root nodes with the smallest weights as the left and right subtrees to construct a new binary tree, and set the weight of the root node of the new binary tree to be the sum of the weights of the root nodes in the left and right trees.
- Delete these two trees from the forest F and add the new binary tree to F.
- Repeat steps 2 and 3 until F contains only one tree.

Repeat steps 2 and 3 until F contains only one tree! [Huffman tree construction process](https://i-blog.csdnimg.cn/blog_migrate/b5e800e7d72923f08448720508d603bf.png#pic_center)

#### 5.2.2 Implementation of the Huffman algorithm

- If there is no node of degree 1 in a Huffman tree, then a Huffman tree with n leaf nodes has 2n-1 nodes, which can be stored in a one-dimensional array of size 2n-1 .

  - Cell 0 of the array is not used, starting from cell 1, so the size of the array is 2n.
    - The leaf nodes are stored centrally in the first 1 to n positions, and the remaining non-leaf nodes are stored in the next n-1 positions.
- Composition

  - Left child
  - Right child
