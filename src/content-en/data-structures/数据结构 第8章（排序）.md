## 1. Overview of basic concepts and sorting methods

## 1.1 Basic concepts of sorting

#### 1.1.1 Sorting

- **Sorting**: The operation of reordering a set of records in non-decreasing or non-increasing order of keywords.

#### 1.1.2 Stability of Sorting

- When the keywords in the sorted records are not the same, the result obtained by sorting the unordered sequence of any one record is unique

#### 1.1.3 Internal and external sorting

- **Internal sorting**: the process of sorting the records to be sorted by storing all of them in the computer memory.
- **External sort**: a sorting process in which the number of records to be sorted is so large that the memory cannot hold them all at once, and access to the external memory is required during the sorting process.

### 1.2 Classification of internal sorting methods

- Sort Record Area

  - Ordered Sequence Area
  - Unordered Sequence Area
- Internal Sorting Methods

  - Insertion Class: "Inserts" one or more records from an unordered subsequence into an ordered sequence, thereby increasing the length of the ordered subsequence of records.
    - Mainly includes:
      - Direct insertion sort
      - Folded Insertion Sort
      - Hill Sort
  - Swap: Increases the length of an ordered subsequence of records by "swapping" one or more records in the unordered subsequence and "inserting" them into the ordered subsequence.
    - Mainly includes:
      - Columnar Sort
      - Quick Sort
  - Selection: Increases the length of an ordered subsequence of records by "selecting" the record with the smallest or largest keyword from the unordered subsequence and adding it to the ordered subsequence.
    - Mainly includes
      - Selection sort
      - Tree Selection Sort
      - Heap sort
  - Merge: gradually increase the length of the ordered subsequence of records by "merging" two or more records into the ordered subsequence.
    - Mainly includes
      - 2-way subsumption sort
  - Allocation class: the only class of sorting methods that do not require keyword comparisons, the sorting is mainly done using two basic operations: allocation and collection.
    - Mainly includes
      - Base Sort

### 1.3 Storage of records to be sorted

- Sequential table: the order relationship between records is determined by their storage locations, and records need to be moved to realize sorting.
- Link list sorting: the order relationship between the records is indicated by the pointers. To realize sorting, it is not necessary to move the records, but only need to modify the pointers.
- Address sorting: the records to be sorted are stored in a set of continuous address storage unit, and at the same time set up an address vector indicating the storage location of each record, in the sorting process, do not move the record itself, but to move the "address" of those records in the address vector, and then adjust the value in the address vector according to the value of the record at the end of sorting. ### 1.4 Evaluation metrics for efficiency of sorting algorithms

- Execution time: for sorting operation, time is mainly consumed in the comparison between keywords and the movement of records, and the time complexity of the sorting algorithm is determined by these two indexes.
- Auxiliary space: in addition to the space occupied by the records to be sorted, other storage space required for the execution of the algorithm, the space complexity is determined by the auxiliary space of the sorting algorithm.

## 2. Insertion Sort

## 2.1 Direct Insertion Sorting

- **Straight Insertion Sort**: Inserts a row into an ordered table, resulting in a new ordered table with an increasing number of rows of one.

```c
void InsertSort(SQList* L) {
	for (int i = 2; i < MAXSIZE + 1; i++)
	{
		if (L->r[i].key < L->r[i-1].key)
		{
			L->r[0] = L->r[i];
			for (int j = 1; j < i; j++)
			{
				if (L->r[0].key < L->r[j].key)
				{
					L->r[i] = L->r[j];
					L->r[j] = L->r[0];
					L->r[0] = L->r[i];
				}
			}
		}
	}
	L->r[0].key = L->r[0].value = 0;
	printf("InsertSort Success\n");
}
```

### 2.2 Folded Insertion Sort

- **Binary Insertion Sort**: a direct insertion sort that uses "half lookup" to find records.

```c
void BInsertSort(SQList* L) {
	for (int i = 2; i < MAXSIZE + 1; i++)
	{
		int low = 1;
		int high = i - 1;
		L->r[0] = L->r[i];							// 将待插入的记录暂存到监视哨中
        
		while (low <= high)							// 在 r[low~high] 中折半查找插入的位置
		{
			int m = (low + high) / 2;				// 折半
			if (L->r[0].key < L->r[m].key)
			{
				high = m - 1;						// 插入点在前一子表
			}
			else
			{
				low = m + 1;						// 插入点在后一子表
			}
		}

		for (int j = i - 1; j >= high + 1; j--)
		{
            L->r[j + 1] = L->r[j];					// 记录后移
		}
		
		L->r[high + 1] = L->r[0];					// 将原 r[i] 插入到正确位置
	}
	L->r[0].key = L->r[0].value = 0;
	printf("BInsertSort Success\n");
}
```

```c
void ShellInsert(SQList* L, int step) {
	for (int i = step + 1; i < MAXSIZE + 1; i++)
	{
		if (L->r[i].key < L->r[i - step].key)		// 将 L->r[i] 插入有序增量子表

			L->r[0] = L->r[i];						// 暂存 L->r[i] 于 L->r[0]

	}
```

## 3. Swap Sort

## 3.1 Colossal ordering

- **Bubble Sort**: compares the keywords of neighboring records and swaps them if they are in reverse order, so that records with small keywords gradually "float" upward like bubbles (left shift), or records with large keywords gradually "fall" downward like stones "falling" (right shift).

```c
void BubbleSort(SQList* L) {
	int flag = 0;									// flag 标记某一趟排序是否发生交换
	int index = MAXSIZE;

	while ((flag == 0) && (index > 1))
	{
		flag = 1;									// flag 置为 1 ，如果本趟没有发生交换，则不会执行下一趟排序

		for (int i = 1; i < index; i++)
		{
			if (L->r[i].key > L->r[i+1].key)
			{
				flag = 0;							// flag 置为 0 ，表示本趟排序发生了交换
				
				L->r[0] = L->r[i];					// 交换前后两个记录
				L->r[i] = L->r[i + 1];
				L->r[i + 1] = L->r[0];
			}
		}

		index--;
	}
	L->r[0].key = L->r[0].value = 0;
	printf("BubbleSort Success\n");
}
```

### 3.2 Quick Sort

- **Quick Sort**: eliminates multiple reverse orders by exchanging two (non-adjacent) records.

```c
int Partition(SQList* L, int low, int high) {
	L->r[0] = L->r[low];							// 用子表的第一个记录做枢轴记录

	int key = L->r[low].key;						// 枢轴记录关键字保存在 key 中
    
	while (low < high)								// 从表的两端交替地向中间扫描
	{
		while ((low < high) && (L->r[high].key >= key))
		{
			high--;
		}

		L->r[low] = L->r[high];						// 将比枢轴小的记录移动到低端

		while ((low < high) && (L->r[low].key <= key))
		{
			low++;
		}

		L->r[high] = L->r[low];						// 将比枢轴大的记录移动到高端

	}
	L->r[low] = L->r[0];							// 枢轴记录到位

	return low;										// 返回枢轴位置
}

void QSort(SQList* L, int low, int high) {
	if (low < high)
	{
		int middle = Partition(L, low, high);		// 将 L->r[low···high] 一分为二，midddle 是枢轴位置
		QSort(L, low, middle - 1);					// 对左子表递归排序
		QSort(L, middle + 1, high);					// 对右子表递归排序
	}
}

void QuickSort(SQList* L) {
	QSort(L, 1, MAXSIZE);
	printf("QuickSort Success\n");
}
```

### 4.3 Heap Sort

- **Heap Sort**: Consider the records to be sorted, r[ 1 --- n ], as the sequential storage structure of a complete binary tree, and utilize the intrinsic relationship between parent and child nodes in the complete binary tree to select the record with the largest (or smallest) keyword in the current unordered sequence.
- Heap: a sequence of n elements

- If the one-dimensional array corresponding to this sequence (with the one-dimensional array as the storage structure for this sequence) is regarded as a complete binary tree, then the heap is essentially a complete binary tree satisfying the following properties

  - The value of all non-terminal nodes in the tree is not greater (or less) than the value of its left and right child nodes.
- Heap sort utilizes the feature that the record at the top of the big-root heap (or little-root heap) has the largest (or smallest) keyword, making it easy to select the record with the largest (or smallest) keyword in the current unordered sequence.

#### 4.3.1 Adjusting the Heap

- Sifting method: sift smaller keywords down layer by layer and select larger keywords up layer by layer

```
void HeapAdjust(SQList* L, int s, int m) {
	L->r[0] = L->r[s];

	for (int i = 2 * s; i <= m; i *= 2)					// 沿 key 较大的孩子结点向下筛选
	{
		if ((i < m) && (L->r[i].key < L->r[i+1].key))	// i 为 key 较大的记录的下标
		{
			i++;
		}

		{
			break;
		}

		L->r[s] = L->r[i];
		s = i;
	}

	L->r[s] = L->r[0];
}
```

## 5. Summarization (problematic)

- **Merging Sort**: the process of combining two or more ordered tables into one ordered table
- **2-way merging**: the process of combining two ordered tables into one ordered table
- Algorithmic Ideas

  - Assuming that the initial sequence contains n records, it can be viewed as n ordered subsequences, each of which is of length 1 , and then merged two by two to get
    $$
    ⌈\frac{n}{2}⌉
    $$
  - Merge two and two, - - - , and so on, until we get an ordered sequence of length n

```c
void Merge(RedType R[], RedType T[], int low, int mid, int high) {
	int i = low;
	int j = mid + 1;
	int k = low;

	while (i <= mid && j <= high )
	{
		if (R[i].key <= R[j].key)
		{
			T[k++] = R[i++];
		}
		else {
			T[k++] = R[j++];
		}
	}

	while (i <= mid)
	{
		T[k++] = R[i++];
	}

	while (j <= high)
	{
		T[k++] = R[j++];
	}
}

void MSort(RedType R[], RedType T[], int low, int high) {
	if (low != high)
	{
		int mid = (low + high) / 2;
		RedType S[MAXSIZE + 1];

		MSort(R, S, mid + 1, high);
		Merge(S, T, low, mid, high);
	}
	else {
		T[low] = R[low];
	}
}
```
