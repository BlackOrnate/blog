## 1. what is studied in data structures

- Data structures is the study of objects that operate in non-numerical computational programming, as well as the relationships and operations between these objects
  - The mathematical model for non-numerical computation problems is no longer mathematical equations, but data structures such as linear tables, trees, and graphs

## 2. Basic concepts and terminology

## 2.1 Data, data elements, data items, and data objects

- **Data**: A symbolic representation of an objective object.
  - A general term for all symbols that can be entered into a computer and processed by a computer program.
- **Data Element (Data Element)**: the basic unit of data.
  - Usually considered and processed as a whole in a computer.
- **Data Item (Data Item) **: the composition of the data element, has an independent meaning, the smallest indivisible units
- **Data Object (Data Object) **: a collection of data elements of the same nature
  - A subset of data

### 2.2 Data Structures

- **Data Structure**: A collection of data elements that have one or more specific relationships with each other.
  - A collection of data elements with a structure
    - Structure: the relationships between data elements
- Includes two levels
  - Logical structure
  - Logical structure

#### 2.2.1 Logical structure

- **Logical structure**: describes the data in terms of logical relationships, it is independent of the storage of the data and is computer independent

  - Can be seen as a mathematical model abstracted from a specific problem
- There are two elements

  - Data elements
  - Relationships
    - Logical relationships between data elements
- Depending on the different characteristics of the relationships between data elements, there are usually four basic types (in increasing order of complexity)

  - Collective structure
    - Collection structure There is no relationship between data elements other than "belonging to the same set".
  - Linear structures
    - One-to-one relationships between data elements
  - Tree structure
    - One-to-many relationships between data elements
  - Graph structure
    - Many-to-many relationships exist between data elements
- Logical structure of data

  - Linear structure
    - Linear tables
      - General Linear Tables
        - Linear Tables: Typical Linear Structures
      - Special Linear Tables
        - Stacks and queues: data operations can be performed only at one or both ends of the table
        - String: data elements consist of only one character
      - Generalizations of Linear Tables
        - Arrays: data elements are a linear table
        - Generalized tables: data elements are a linear table, but not disjoint, i.e., can be a single element or a linear table
  - Non-linear structures
    - Tree structure
      - Tree: a hierarchical structure with multiple branches
      - Binary tree: a hierarchical structure with two branches.
    - Graph structure
      - Directed graph: ordered tree whose edges are vertices
      - Undirected graph: an unordered tree whose edges are vertices.
    - Collection structure

#### 2.2.2 Storage Structures

- **Storage structure**: the storage representation of a data object in a computer

  - Also known as **physical structures**
- Two basic data structures

  - Sequential storage structure

    - Stored in a contiguous piece of storage space.
    - Logical relationships between data elements are represented by their relative positions in memory.
    - Usually described by the array type of a programming language.
  - Chained storage

    - Does not require an entire block of memory.
    - In order to represent the relationships between nodes, each node needs to have a pointer field attached to it, which holds the storage address of the succeeding element.
    - Usually expressed with the help of the pointer type of a programming language.
    - Each node occupies two consecutive memory cells.
      - One holds information about the node.
      - One holds information about the node and the other holds the first address of the successor node.

### 2.3 Data Types and Abstract Data Types

#### 2.3.1 Data Types

- **Data Type**: a basic concept in high-level programming languages
- In a programming language, every piece of data belongs to a data type.
  - A type explicitly or implicitly specifies the range of values, the way data is stored, and the operations that are allowed, **Data Type** is a general term for a set of values and a set of operations defined on that set of values.
- Programming language allows users to directly use the data types determined by the specific language, data types reflect the data description and processing capabilities of the programming language
- C language in addition to providing integer, real, character type and other basic types of data, but also allows users to customize a variety of types of numbers
  - arrays, structures, pointers, etc.

#### 2.3.2 Abstract Data Types

- Abstraction: extracting the essence of the actual problem

  - Advanced data abstraction
    - Data types
      - Integer, real, character, etc.
    - The above types can be used to construct
      - Complex abstract data types such as linear tables, stacks, queues, trees, etc. can be constructed using the above types.
- Abstract Data Type (ADT)**: a user-defined mathematical model that represents an application problem and a set of operations defined on that model.

  - Contains 3 parts

    - Data object
    - A collection of relationships on the data object
    - A collection of basic operations on the data object
  - Definition format

    ```
    ADT 抽象数据类型名 {
        数据对象：<数据对象的定义>
        数据关系：<数据关系的定义>
        基本操作：<基本操作的定义>
    } ADT 抽象数据类型名
    ```

    - Definition of data objects and data relationships using mathematical notation and natural language descriptions.
    - Definition format of basic operations

      ```
      基本操作名(参数列表)
          初始条件:<初始条件描述>
          操作结果:<操作结果描述>
      ```

      - The basic operation has two parameters
        - Assignment parameter: only provide input value for the operation.
        - Reference parameter: starts with &, in addition to providing the input value, it will also return the result of the operation.
      - Initial conditions: the conditions that the data structure and parameters should meet before the operation is executed.
        - If the initial condition is null, it is ignored.
      - Result: after the operation completes normally, the transformation of the data structure and the result to be returned.

## 3. Algorithms and Algorithm Analysis

## 3.1 Definition and Characteristics of Algorithms

- Algorithm: a finite sequence of operations specified to solve a certain type of problem.
- 5 important properties

  - **Infiniteness**:

    - An algorithm must always end after performing an infinite number of steps, and each step must be completed in an infinite amount of time
  - **Determinism**:

    - The operations to be performed in each case are specified in the algorithm, without duality, and are clear to the performer or reader of the algorithm in terms of what they mean and how they are performed.
  - **Feasibility**:

    - All operations in the algorithm can be realized by executing the basic operations already implemented a finite number of times.
  - **Input**:

    - An algorithm has 0 or more outputs
    - When describing algorithms in terms of functions, the inputs are often represented by formal parameters, which get their values from the main function when they are called
  - **OUTPUT**:

    - An algorithm has 1 or more outputs, which are the result of the information processing performed by the algorithm; an algorithm with no outputs has no meaning
    - When describing an algorithm as a function, the outputs are mostly in the form of return values or formal parameters of reference type

### 3.2 Basic Criteria for Evaluating the Merits of Algorithms

- 4 aspects of evaluation
  - **Correctness**:
    - Ability to obtain correct results in limited runtime with reasonable data inputs
  - **Readability**:
    - Readable algorithms help people to understand them, while difficult algorithms tend to hide errors and are hard to debug and modify
  - **Robustness**:
    - When the input data is illegal, a good algorithm can appropriately react correctly or process accordingly without producing some inexplicable output results
  - **Efficiency**:
    - Includes 2 aspects
      - Time
        - Algorithms are well-designed and efficiently executed, which can be measured in terms of time complexity
      - Space
        - Spatial aspect The algorithm occupies a reasonable amount of storage capacity, which can be measured in terms of space complexity.
      - Time complexity and space complexity are the two main indicators of algorithms

#### 3.3.3 Example of Time Complexity Analysis of Algorithm

- The basic method of analyzing the time complexity of an algorithm:
  - Find the statement with the highest frequency among all statements as the basic statement, calculate the frequency of the basic statement to get the function f(n) of the problem size n, and take its order of magnitude to be expressed by "O".

##### Theorem 1.1

$$
若f(n) = a_mn^m + a_{m-1}n^{m-1} + \cdots + a_1n + a_0是一个m次多项式，则T(n)=O(n^m)
$$

- Constant Order Example

  ```
  i++;
  j=0;
  ```

  - If both statement frequencies are 1, the execution time of the algorithm is a constant independent of the problem size n, so the algorithm has a time complexity of T(n) = O(1), which is called constant order.
  - If the execution time of the algorithm does not increase as the problem size n increases, the frequency of statements in the algorithm is a constant
    - Even if this constant is large, the time complexity of the algorithm is O(1)
- Linear Order Example

  ```
  for (i = 0; i < n; i++) {
      x++;
      j = 0;
  }
  ```

  - The frequency of both statements within the loop is f(n) = n, so the time complexity of the algorithm is T(n) = O(n), called linear order
- Example of squared order

  ```
  x = 0;
  y = 0;

  for (i = 1; i <= n; i++) {
      x++;

  for (i = 1; i <= n; i++) {
      for (j = 1; j <= n; j++) {
          y++;
      }
  }
  ```

  - The frequency is f(n) = n^2, so the time complexity of the algorithm is T(n) = O(n^2), called square order

#### 3.3.4 Best, Worst and Average Time Complexity

- **Best time complexity**: the time complexity of the algorithm in the best case scenario
  - Minimum possible amount of computation for the algorithm
- **Worst time complexity**: time complexity of the algorithm in the worst case scenario
  - Maximum possible value of the algorithm's computation
- **Average time complexity**: the weighted average of the algorithm's computation in all possible cases when the inputs actually occur with equal probability according to the inputs

```python

- Best time complexity: f(n) = 1
- Worst time complexity: f(n) = n
- Average time complexity: f(n) = n/2
