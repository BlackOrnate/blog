## 1. Basic concepts of lookup

- **Finding table**: a collection of data elements (or records) of the same type
- **Keyword**: the value of a data item in a data element (or record) with which a data element (or record) can be identified
  - **Primary keyword**: a keyword that uniquely identifies a record
  - **Sub-keyword**: a keyword that identifies a number of records
- **Find**: based on a given value, identify a record or data element in the lookup table whose keyword is equal to the given value
  - **Find Successful**: the existence of such a record in the table, the result of the search can give information about the entire record, or indicate the location of the record in the lookup table
  - ** lookup unsuccessful **: the table does not exist in the keyword is equal to the given value of the record, this time the results of the lookup can give a "null" record or "null" pointer
- **Dynamic lookup table**: lookup at the same time to do the modification operation (insertion or deletion) of the table
- **Static lookup table**: a table used only for lookups
- **Average Search Length (ASL)**: the position of a record in a lookup table (the expected value of the number of keywords compared to a given value)

## 2. Linear Table Lookup

## 2.1 Sequential Lookup

- **Sequential Search**: Starting from one end of the table, the keywords of the rows are compared with the given value in turn.
  - If the keyword of a record is equal to the given value, the search is successful.
  - If no record has a keyword equal to the given value, the search fails.

```c
void Search_Seq1(SqList L, int first) {
	for (int i = 0; i < L.length; i++)

		if (L.abc[i].first == first)
		{
			printf("i = %d\n", i);
			printf("Search Success\n");
			return 0;
		}
	}
	printf("Search Error\n");
}
```

- Both algorithms have the same time complexity

  $$
  ASL = \frac{1}{n} \sum_{i=1}^{n}{i} = \frac{n+1}{2}
  $$
   Time complexity is O(n)
- Advantages: simple algorithm, no requirements on table structure, applicable to sequential structure, also applicable to chained structure, can be applied regardless of whether the records are ordered by keywords or not
- Disadvantages: the average search length is large, the search efficiency is low

### 2.2 Folded Half Lookup

- Binary Search (binary search) **: the linear table must use the sequential storage structure, and the elements in the table are ordered according to the keywords.
  - Starting from the middle record of the table, if the given value and the keyword of the middle record are equal, then the search is successful.
  - If the given value is greater or less than the keyword of the middle record, then search in the half of the table that is greater or less than the middle record.
  - Repeat until the search succeeds, or if the search area is empty at one step, the search fails.

```c
void Search_Bin(SqList L, int first) {
	int low = 0;
	int high = L.length - 1;
	int mid = 0;

	while (low <= high)
	{
		mid = (low + high) / 2;
		if (first == L.abc[mid].first)
		{
			printf("mid = %d\n", mid);
			printf("Search Success\n");
			return 0;
		}
		else if (first < L.abc[mid].first)
		{
			high = mid - 1;
		}
		else {
	
			low = mid + 1;
		}
	}
	printf("Search Error\n");
}
```

### 2.3 Finding in Chunks

- **Blocking Search (indexed sequential lookup)**: introduces an "indexed table" in addition to the table itself

  - Divide the records in the table into sub-tables, and create an index for each sub-table (block)
  - Index contents
    - Keyword entries (the largest keyword within that sub-table)
    - Pointer item (first record of that sub-table in the table position)
  - The indexed table is ordered by keyword, and the table is either ordered or chunked.
    - Chunked ordering: all records in the second sub-table have a keyword greater than the largest keyword in the first sub-table, all records in the third sub-table have a keyword greater than the largest keyword in the second sub-table, and so forth

  $$
  ASL_{bs} = L_b + L_w = \frac{1}{b} \sum_{j=1}^{b}{j} + \frac{1}{s} \sum_{i=1}^{s}{i} = \frac{b+1}{2} + \frac{s+1}{2} = \frac{1}{2} (\frac{n}{s} + s) + 1
  $$

## 3.1 Binary Sorted Trees (added later)

- **Binary Sort Tree (binary lookup tree)**: special binary tree useful for both sorting and lookups

! [Binary Sort Tree Example](https://i-blog.csdnimg.cn/blog_migrate/169e73f794e78e407eb046842a87b096.png#pic_center)

#### 3.1.1 Definitions

- If the left subtree of a binary sorted tree is not empty, then all nodes in the left subtree have values less than the value of its root node.
- If the right subtree of a binary sorted tree is not empty, then all nodes in the right subtree are greater than the value of its root node.
- The left and right subtrees of a binary sorted tree are also binary sorted trees
- Intermediate traversal of a binary sorted tree results in an ordered sequence of nodes with increasing values

```c
typedef struct {
	int first;										// 关键字项
	int second;										// 其他数据项
} ABC;												// 每个结点的数据域的类型

typedef struct {
	ABC abc;										// 每个结点的数据域
	struct BSTNode* lchild, * rchild;				// 左右孩子指针
} BSTNode, *BSTree;
```

#### 3.1.2 Finding

- Steps
  - If the binary tree is empty, the search fails and the null pointer is returned.
  - If the binary tree is not empty, compare the given value key with the key of the root node, T->data.key.
    - If key is equal to T->data.key, the search succeeds and the root node address is returned.
    - If key is less than T->data.key, find left subtree recursively.
    - If key is greater than T->data.key, find right subtree recursively.

```python
BSTree SearchBST(BSTree T, int value) {
	if ((!T) || value == T->abc.first)
	{
		return T;
	}
	else if (value < T->abc.first) {

		return SearchBST(T->lchild, value);
	}
	else {
		return SearchBST(T->rchild, value);
	}
}
```

#### 3.1.3 Insertion

- Procedure
  - If the binary tree is empty, the node to be inserted \*S is inserted into the empty tree as the root node.
  - If the binary tree is not empty, compare the key with the key of the root node, T->data.key.
    - If key is less than T->data.key, insert \*S into left subtree
    - If key is greater than T->data.key, insert \*S into right subtree

```c
void InsertBST(BSTree* T, ABC abc) {
	if (!T)
	{
		BSTNode* S = NULL;
		S = (BSTNode*)malloc(sizeof(BSTNode));
		S->abc = abc;
		S->lchild = S->rchild = NULL;
		T = S;
	}
	else if (abc.first < (*T)->abc.first) {
		InsertBST((*T)->lchild, abc);
	}
	else if (abc.first > (*T)->abc.first) {
		InsertBST((*T)->rchild, abc);
	}
}
```

#### 3.1.4 Creating

- Steps
  - Initialize the binary sort tree T as an empty tree.
  - Read in a node with the keyword key.
  - If the keyword key is not an end-of-entry flag, then do the following
    - Insert this node into the binary tree T.
    - Read in a node whose key is key

```c
int Delete(BiTree *p)
{
    BiTree q, s;
    if(!(*p)->lchild && !(*p)->rchild){
        *p = NULL;
    }
    else if(!(*p)->lchild){
        q = *p;
        *p = (*p)->rchild;
        free(q);
    }
    else if(!(*p)->rchild){
        q = *p;
        *p = (*p)->lchild;
        free(q);
    }
    else{
        q = *p;
        s = (*p)->lchild;
        while(s->rchild)
        {
            q = s;
            s = s->rchild;
        }
        (*p)->data = s->data;
        if( q != *p ){
            q->rchild = s->lchild;
        }else{
            q->lchild = s->lchild;
        }
        free(s);
    }
    return TRUE;
}

int DeleteBST(BiTree *T, int key)
{
    if( !(*T)){
        return FALSE;
    }
    else
    {
        if( key == (*T)->data ){
            Delete(T);
            return TRUE;
        }
        else if( key < (*T)->data){
            return DeleteBST(&(*T)->lchild, key);
        }
        else{
            return DeleteBST(&(*T)->rchild, key);
        }
    }
}
```

### 3.3 B-tree (late addition)

- B-tree: balanced multinomial tree suitable for outer lookups

#### 3.3.1 Definitions

- A B-tree of order m, m-forks

  - Each node in the tree has at most m subtrees
  - if the root node is not a leaf node, then there are at least two subtrees
  - All nonterminal nodes except the root have at least ⌈m/2⌉ subtrees
  - All leaf nodes appear on the same level and carry no information, often called failure nodes
    - Failure nodes do not exist; pointers to these nodes are null
    - Failure nodes are introduced to facilitate the analysis of B-tree lookup performance.
  - All non-terminal nodes have at most m-1 keywords.
- Characteristics

  - Balanced: all leaf nodes are at the same level
  - Ordered: the keywords in each node of the tree are ordered, and the keywords in the "left subtree" of the keyword K\_i are all less than K\_i, while the keywords in the "right subtree" of the keyword K\_i are all greater than K\_i.
  - Multiplex: except for leaf nodes, some nodes have one keyword and two subtrees, and some nodes have two keywords and three subtrees

### 3.4 B+ Tree

- A B+ tree is a variant of a B-tree.

#### 3.4.1 Differences between B+ and B-tree

- There are n subtrees with n nodes containing n keywords.
- All leaf nodes contain all the information of the keywords and pointers to the records containing these keywords, and the leaf nodes themselves are linked in descending order of the size of the keywords.
- All non-terminal nodes can be viewed as indexed sections containing only the largest (or smallest) keyword in their subtree (the root node).

#### 3.4.2 Finding

- If the keyword on a non-terminal node is equal to a given value, it does not terminate, but continues down until the leaf node

#### 3.4.3 Insert

- Insertion is performed only on leaf nodes, and when the number of keywords in a node is greater than m it is split into two nodes, which contain the number of keywords ⌊(m+1)/2⌋ and ⌈(m+1)/2⌉ respectively.
- Their biparent nodes should contain the largest keyword in both nodes

#### 3.4.4 Deletion

- Deletion of a B+ tree also takes place only at leaf nodes, and when the largest keyword in a leaf node is deleted, its value in a non-terminal node can exist as a "dividing keyword"
- If the number of keywords in a node is less than ⌈m/2⌉ due to deletion, the merging process with its sibling nodes is similar to that of a B-tree.

## 4. Finding Hash Tables

## 4.1 Concepts

- **Hash Search (Hash Search) (Hash, Hash)**: through the element of the keyword value of some kind of operation, the direct address of the element, i.e., using the keyword to address of the direct conversion method, do not need to be repeated comparisons
- **Hash address**: the storage location of the record
- **Hash function**: a defined correspondence between a record's storage location and its keyword.
- **Hash table**: a finite contiguous address space for storing data records whose corresponding hash addresses are calculated by the hash function.

  - Usually the storage space of a **hash table** is a one-dimensional array, and the hash address is the subscript of the array.
- **Conflict**: get the same hash address for different keywords
- **Synonyms**: keywords with the same function value

### 4.2 Constructor methods

#### 4.2.1 Numerical analysis methods

- Prior to know the set of keywords, and the number of bits of each keyword is more than the number of bits of the address code of the hash table, each keyword consists of n bits, such as k\_1 k\_2 --- k\_n, then you can extract a number of bits from the keywords with a more uniform distribution of numbers as the hash address
- Application: all the keywords must be clearly known in advance on each bit of the distribution of various figures

#### 4.2.2 Squared center method

- A number after the square of the middle of the number of digits and the number of each are related, if the keyword square the middle of the number of digits or a combination of them for the hash address, so that the random distribution of keywords to get the hash address is also random, the specific number of digits obtained by the table length decision
- Application: can not know all the circumstances of the keyword in advance, or difficult to find the keyword directly from the value of the more scattered bits

#### 4.2.3 Folding method

- Split the keyword into several parts with the same number of digits (the last part of the number of digits can be different), and then take the sum of these parts (rounding) as the hash address.

  - Bitwise superposition: align the lowest bits of each part after division, and then add them together
  - Boundary superposition: fold two neighboring parts back and forth along the boundary, and then add them together
- Application: the number of bits in the hash address is small, while the number of bits in the keyword is large, and it is difficult to find the more scattered bits directly from the keyword.

#### 4.2.4 Dividing by the Remainder Method

- Assuming that the length of the hash table is m, choose a number p that is not greater than m, use p to remove the keyword, and the remainder obtained after division is the hash address.
- The remainder after division is the hash address, which guarantees that the hash address will fall in the address space of the hash table.

### 4.3 Methods for handling conflicts

#### 4.3.1 Open address method

- Basic idea:

  - Store the record in the hash table, when the initial hash address H\_0 = H(key) of the key of a record conflicts, take H\_0 as the basis, take the appropriate method to calculate to get another address H\_1, if H\_1 is still conflicting, use H\_1 as the basis to find the next address H\_2, if H\_2 is still conflicting, then H\_3 and so on until H\k is not conflicting, H\_k is the hash address of that record in the table. And so on, until H\_k does not conflict, then H\_k is the hash address of the record in the table
- Open address method: in the search for the "next" empty hash address, the original array space is open to all elements
- ** detection **: the process of finding the "next" empty space

  $$
  H_i = (H(key) + d_i) \% m \quad i = 1,2,···,k (k \leq m-1)
  $$
   H(key) : hash function; m : hash table table length , d\_i : incremental sequence

- **Secondary aggregation (stacking)**: the phenomenon of two records with different first hash addresses competing for the same successor hash address that occurs during conflict handling
- Linear detection method

  - Advantage: as long as the hash table is not filled, it is always possible to find a non-conflicting address
  - Disadvantage: "secondary aggregation" occurs.
- Secondary and pseudo-random probing methods

  - Advantage: avoids "secondary aggregation".
  - Disadvantage: no guarantee of finding a non-conflicting address

#### 4.3.2 Chaining Addresses

- Basic idea:
  - Synonym chain table: put the records with the same hash address in the same single chain table.
  - If there are m hash addresses, there are m single linked lists, meanwhile, HT[ 0 --- m-1 ] is used to store the head pointer of each linked list, and all the records with hash address i are inserted into the single linked list with HT[i] as the head node in a node way.

HT[i] is the head node of the single chained table. [Hash table for conflict handling by chain address method](https://i-blog.csdnimg.cn/blog_migrate/60b4a72bd98916a9d528fddc6ef00d15.png#pic_center)

- Storing the hash table

```c
#define m 10										// 散列表的表长

typedef struct {
    KeyType key;									// 关键字项
```

- Steps
  - Given the keyword to be searched, calculate H\_0 = H(key) according to the hash function set during table construction.
  - If cell H\_0 is empty, the element to be searched does not exist.
  - If the keyword of the element in cell H\_0 is key, then the search is successful.
  - Otherwise, repeat the following conflict resolution procedure
    - Calculate the next hash address, H\_i, in the same way as for conflict resolution.
    - If cell H\_i is empty, the element to be searched does not exist.
    - If cell H\_i is empty, the element to be searched does not exist If the keyword of the element in cell H\_i is key, the search is successful

```c
void SearchHash(HashTable HT, KeyType key) {
    int H_0 = H(key);
    for (int i = 0; i < m; i++) {
        int H_i = (H_0 + i) % m;
        if (HT[H_i].key == 0) {
            printf("Not Exist");
            return 0;
        } else if (HT[H_i].key == key){
            printf("Search Success");
            return 0;
        }
    }
}
```

- Algorithm Analysis

  - The **loading factor α** of a hash table is defined as

    $$
    \alpha = \frac{表中填入的记录数}{散列表的长度}
    $$
  - Average lookup length of the hash table for different methods of handling conflicts with equal probability

    - Linear detection method

      - Successful search

        $$
        \frac{1}{2} (1 + \frac{1}{1-\alpha})
        $$
      - Failed.

        $$
        \frac{1}{2} [1 + \frac{1}{(1-\alpha)^2}]
        $$
    - Secondary detection, pseudo-random detection

      - Find Success

        $$
        -\frac{1}{\alpha} ln(1-\alpha)
        $$
      - Failed.

        $$
        \frac{1}{1-\alpha}
        $$
    - Chain address method

      - Find Success

        $$
        1 + \frac{\alpha}{2}
        $$
      - Failed

        $$
        \alpha + e^{- \alpha}
        $$
        α+e-α
  - With equal probability, the average lookup length is computed directly

    - Successful search

      $$
      ASL_{succ} = \frac{1}{n} \sum_{i=1}^{n}{C_i}
      $$
       n: the number of records in the hash table; C\_i: the number of comparisons required to successfully find the ith record.
    - Failure

      $$
      ASL_{unsucc} = \frac{1}{r} \sum_{i=1}^{r}{C_i}
      $$
       n: the number of hash function values; C\_i: the number of comparisons that failed to find the hash function when it took value i
