## 1. The basics of concurrency

### 1.1 Case introduction

- https://www.httpbin.org/delay/5
- The server is forced to wait 5 seconds for a response to be returned

```python
import logging
import time
import requests

logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s %(levelname)s: %(message)s')

URL = "https://www.httpbin.org/delay/5"
TOTAL_NUMBER = 10

start_time = time.time()

for _ in range(1, TOTAL_NUMBER):
    logging.info(f"scraping {URL}")
    response = requests.get(URL)

end_time = time.time()
logging.info(f"total time {end_time - start_time}")
```

### 1.2 Basics

#### Blocking

- Blocking: the state in which a program hangs when it does not get the computing resources it needs
- A program is operationally blocked: the program cannot continue to do anything else while it waits for an operation to complete.

#### Non-blocking

- A program is operationally non-blocking: while waiting for an operation to complete, the program is not blocked and can continue to do other things.
- A program may be non-blocking only if it is encapsulated at a level that can encompass independent subroutine units
- Exists because of the existence of blocking

#### synchronization

- Synchronized execution of program units: the need for different program units to communicate in order to accomplish a task together

#### Asynchronous

- In order to accomplish a certain task, sometimes different program units can accomplish the task without the need for communication and coordination

#### Multi-process

- Taking advantage of the multiple cores of the CPU, multiple tasks can be executed in parallel at the same time, which can greatly improve the execution efficiency.

#### Concurrent Programs

- A lightweight thread that runs in the userland.
- Has its own register context and stack.
- Scheduling switch
  - Save the register context and stack elsewhere, and restore the saved register context and stack when you cut back.
- Can retain the state of the last call, a particular combination of all local states, so that each time the procedure reenters, it enters the state of the last call.
- Essentially a single thread
- Implementing asynchronous operations

### 1.3 Usage of Concurrency

- The asyncio library
  - event\_loop:
    - Event loop.
    - Register a function to this event loop and call the corresponding handler when the occurrence condition is met.
  - coroutine:
    - Coroutine.
    - Represents the type of the coroutine object.
    - A coroutine object can be registered to a time loop, and it will be called by the event loop.
  - task:
    - task.
    - A further encapsulation of the concurrent object.
      - Contains the state of the concatenation object.
  - future:
    - The result of a task that will or will not be executed.
    - Not fundamentally different from task
  - async keyword:
    - Defines a method (a concurrent program).
      - This method is not executed immediately when it is called, but returns a concurrent object.
  - await keyword:
    - Hangs and blocks the execution of the method.

### 1.4 Defining a concurrent method

```python
import asyncio

async def execute(x):
    print(f"Number: {x}")

coroutine = execute(1)
print(f"Coroutine: {coroutine}")
print("After calling execute")

loop = asyncio.get_event_loop()
loop.run_until_complete(coroutine)
print("After calling loop")
```

- When you pass the coroutine object to the run\_until\_complete method, you actually perform an action
  - Encapsulate the coroutine into a task object.
- Display the task declaration.

### 1.5 Binding Callbacks

- Bind a callback method to the task object

```python
import asyncio
import requests

async def request():
    url = "https://www.baidu.com/"
    status = requests.get(url)
    return status

def callback(task):
    print(f"Status: {task.result()}")

coroutine = request()
task = asyncio.ensure_future(coroutine)
task.add_done_callback(callback)
print(f"Task: {task}")

loop = asyncio.get_event_loop()
loop.run_until_complete(task)
print(f"Task: {task}")
```

## 2. use of aiohttp

### 2.1 Basic introduction

- asyncio: asynchronous operation of TCP, UDP and SSL protocols.
- aiohttp: asynchronous operation of HTTP requests.
- aiohttp is an asynchronous HTTP network module based on asyncio
  - Provides both server-side and client-side
    - server-side
      - You can build a server that supports asynchronous processing
        - used to process the request and return the response
      - Similar to Django, Flask, Tornado and some other Web servers
    - Client
      - Initiating a request
      - Similar to using request to initiate an HTTP request and get a response.
        - request initiates a synchronous web request
        - aiohttp initiates an asynchronous web request.

### 2.2 Basic Examples

```python
import aiohttp
import asyncio

async def fetch(session, url):
    # 上下文管理器，自动分配和释放资源
    async with session.get(url) as response:
        return await response.json(), response.status

async def main():
    # 上下文管理器，自动分配和释放资源
    async with aiohttp.ClientSession() as session:
        url = "https://www.httpbin.org/delay/5"
        html, status = await fetch(session, url)
        print(f"html: {html}")
        print(f"status: {status}")

if __name__ == "__main__":
    loop = asyncio.get_event_loop()
    loop.run_until_complete(main())
    # 较高版本的python可以不显示声明事件循环
    asyncio.run(main())
```

### 2.8 Concurrency Limits

- Using Semeaphore to control concurrency
  - Prevent websites from becoming unresponsive for a short period of time
  - Avoiding the risk of hanging the target site from crawling

```python
import aiohttp
import asyncio

# 爬取的最大并发量
CONCURRENCY = 5
URL = "https://www.baidu.com/"

# 创建一个信号量对象
semaphore = asyncio.Semaphore(CONCURRENCY)
session = None

async def scrape_api():
    # 信号量可以控制进入爬取的最大协程数量
    async with semaphore:
        print(f"Scraping {URL}")
        async with session.get(URL) as response:
            await asyncio.sleep(1)
            return await response.text()

async def main():
    global session
    session = aiohttp.ClientSession()
    scrape_index_tasks = [
        asyncio.ensure_future(
            scrape_api()) for _ in range(1000)]
    await asyncio.gather(*scrape_index_tasks)

if __name__ == "__main__":
    asyncio.run(main())
```

```python
import json
import asyncio
import logging
import aiohttp

logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s %(levelname)s: %(message)s')

INDEX_URL = "https://spa5.scrape.center/api/book/?limit=18&offset={offset}"
DETAIL_URL = "https://spa5.scrape.center/api/book/{id}"
PAGE_SIZE = 18
PAGE_NUMBER = 100
CONCURRENCY = 5

semaphore = asyncio.Semaphore(CONCURRENCY)
session = None

async def scrape_api(url):
    async with semaphore:
        try:
            logging.info(f"Scraping: {url}")
            async with session.get(url, verify_ssl=False) as response:
                return await response.json()
        except aiohttp.ClientError:
            logging.info(f"Error: {url}", exc_info=True)

async def scrape_index(page):
    url = INDEX_URL.format(offset=PAGE_SIZE * (page - 1))
    return await scrape_api(url)

async def scrape_detail(id):
    url = DETAIL_URL.format(id=id)
    data = await scrape_api(url)
    logging.info(f"Saving {url}: {data}")

async def main():
    global session
    session = aiohttp.ClientSession()
    scrape_index_tasks = [
        asyncio.ensure_future(
            scrape_index(page)) for page in range(
            1, PAGE_NUMBER + 1)]
    results = await asyncio.gather(*scrape_index_tasks)
    logging.info(
        f"Results: {json.dumps(results, ensure_ascii=False, indent=2)}")

    ids = []
    for index_data in results:
        if not index_data:
            continue
        for item in index_data.get("results"):
            ids.append(item.get("id"))

    scrape_detail_tasks = [
        asyncio.ensure_future(
            scrape_detail(id)) for id in ids]
    await asyncio.wait(scrape_detail_tasks)
    await session.close()

if __name__ == "__main__":
    asyncio.run(main())
```
