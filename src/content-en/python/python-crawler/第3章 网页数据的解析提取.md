## 1. Using XPath

- **XPath (XML Path Language)
  - XML Path Language
  - Used to find information in XML documents
    - This also applies to html documents.

### 1.1 XPath Common Rules

| Expression | Description |
| --- | --- | nodename | selects all children of this node | all children of this node
| nodename | Select all children of this node | / | Select direct children from current node | / | Select direct children from current node.
| / | | Pick direct children from the current node |
| // | Selects descendants from the current node | | .
| . | ... | Select current node | ...
| ... | Select the parent of the current node |
| @ | Select Attributes | ... | Select the parent of the current node | ...

- Example

```
//title[@lang="eng"]
```

### 1.10 Attribute multi-value matching

- The li attribute has two values
  - li
  - li-first
- Matching with the previous attribute will not get the

```python
from lxml import etree

text = """

"""

```

- Use contains for multi-value matching of attributes

```python
from lxml import etree

text = """
<li class="li li-first"><a href="link.html">first item</a></li>
"""
html = etree.HTML(text)
result = html.xpath("//li[contains(@class, \"li\")]/a/text()")

```

### 1.11 Multi-attribute matching

```python
from lxml import etree

text = """
<li class="li li-first" name="item"><a href="link.html">first item</a></li>
"""
html = etree.HTML(text)
result = html.xpath(
    "//li[contains(@class, \"li\") and @name=\"item\"]/a/text()")
print(result)
```

### 1.12 Sequential Selection

```python
from lxml import etree

html = etree.parse("./test.html", etree.HTMLParser())

# 选取第一个节点
result = html.xpath("//li[1]/a/text()")
print(result)

# 选取最后一个节点
result = html.xpath("//li[last()]/a/text()")
print(result)

# 获取限定位置的节点
result = html.xpath("//li[position() < 3]/a/text()")
print(result)

# 获取倒数第三个节点
result = html.xpath("//li[last() - 2]/a/text()")
print(result)
```

### 1.13 Node axis selection

```python
from lxml import etree

html = etree.parse("./test.html", etree.HTMLParser())

# ancestor轴: 获取所有祖先节点
result = html.xpath("//li[1]/ancestor::*")
print(result)
result = html.xpath("//li[1]/ancestor::div")
print(result)

# attribute轴: 获取所有属性值
result = html.xpath("//li[1]/attribute::*")
print(result)

# child轴: 获取所有直接子节点
result = html.xpath("//li[1]/child::a[@href=\"link1.html\"]")
print(result)

# descendant轴: 获取所有子孙节点
result = html.xpath("//li[1]/descendant::span")
print(result)

# following轴: 获取当前节点之后的所有节点
result = html.xpath("//li[1]/following::*[2]")
print(result)

# following-sibling轴: 获取当前节点之后的所有同级节点
result = html.xpath("//li[1]/following-sibling::*")
print(result)
```

## 2. Using Beautiful Soup

- Parsing web pages by their structure and attributes

### 2.1 The parser

- Beautiful Soup relies on parsers for parsing.
- Parsers supported by Beautiful Soup

| Parsers | Usage | Advantages | Disadvantages
| --- | --- | --- | --- | --- | ---
| Python Standard Library | BeautifulSoup(markup, "html.parser") | Python's built-in standard library, moderately fast execution speeds, good document fault tolerance | Python 2.7.3 or Python 3.2.2 before Chinese fault tolerance Poor Chinese fault tolerance | LXML HTML parser
| LXML HTML parser | BeautifulSoup(markup, "lxml") | Fast, document fault tolerance | Requires C library | LXML XML parser | BeautifulSoup(markup, "lxml") | Fast, document fault tolerance | Requires C library
| LXML XML Parser | BeautifulSoup(markup, "xml") | Fast, only parser that supports XML | Requires C library | html5lib
| html5lib | BeautifulSoup(markup, "html5lib") | Provides the best fault tolerance, parses documents the way they would be parsed in a browser, generates HTML5 formatted documents | Slow, no dependency on external extensions |

### 4.5 Regular extraction

```python
from parsel import Selector

html = """
<div>
    <ul>
        <li class="item-0">first item</li>
        <li class="item-1"><a href="link2.html">second item</a></li>
```
