# 1. URL Components Explained

The basic format of a URL

```
scheme://[username:password@]hostname[:port][/path][;parameters][?query][#fragment]
```

- **scheme(protocol)**: protocol

  - Common protocols: hrrp, https, ftp, etc.
- **username, password**: username, password

  - In some cases, the URL requires a username and password to access it.
- **hostname**: host address

  - Can be a domain name or IP address
- **port**: port

  - _```
    http://localhost:8080
    ```_: port set by the server
  - Port set by the server
- **path**: path

  - Specified address of the network resource in the server
- **parameters**: parameters

  - Used to specify additional information when accessing a resource
- **query**: query

  - Used to query a certain type of resource
  - If there is more than one query, separate them with &
- **fragment**: fragment

  - A partial addition to a resource description
  - Two main applications
    - Single-page routing: Vue and React can use this to manage routing.
    - HTML anchor: control a page to automatically scroll down to a specific position when it opens.

# 4. Two ways to pass parameters to a URL

## 4.1 Querying by String

- `book/views.py`

  ```
  from django.http import HttpResponse

  def book_detail_query_string(request):  # 通过字段查询 http://localhost:8000/book?id=3
      book_id = request.GET.get("id")  # 等同于 request.GET["id"] 但是 get 方法更加安全，当属性不存在时会返回 None 而不是抛出异常
      return HttpResponse(f"book id: {book_id}")
  ```
- `urls.py`

  ```
  from book import views

  urlpatterns = [
      path("book", views.book_detail_query_string),
  ]
  ```

## 4.2 Carrying in path

- `book/views.py`

  ```
  from django.http import HttpResponse

  def book_detail_path_variable(request, book_id):  # 在 path 中携带参数 http://localhost:8000/book/3
      return HttpResponse(f"book id: {book_id}")
  ```
- `urls.py`

  ```
  from book import views

  urlpatterns = [
      path("book/<int:book_id>", views.book_detail_path_variable),  # 添加数据类型限制输入，并限定views.py接收到的book_id为整形
  ]
  ```

  ``` The path function in detail

```
path(route, view, kwargs=None, name=None)
```

- **route:** string defining the path portion of the URL
  - Can contain variables, such as `<int:my_variable>`, to capture parameters from the URL and pass them to the view function
  - Common Data Types
    - `str`: non-empty string (**default**) without slashes (`/`)
    - `int`: non-negative integer
    - __KEEP__34__: strings consisting of a crossbar (__KEEP__35__) or underscore (__KEEP__36__) with characters or numbers
    - `uuid`: `uuid` string of characters
    - `path`: __Non-empty string
- **view: ** view function that handles requests matching a given route
  - Can be a function or a class-based view
- **kwargs (optional):** a dictionary containing additional keyword arguments passed to the view function
- **name (optional):** Assigns a unique name to the URL route so that it can be referenced elsewhere in the code.
  - This is useful for things like generating URLs in templates or redirecting them in code!
