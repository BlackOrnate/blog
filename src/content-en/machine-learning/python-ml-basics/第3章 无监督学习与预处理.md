# 1. Types of unsupervised learning

- Two types of unsupervised learning
  - **Data set transformations (unsupervised transformations of data sets)
    - Algorithms that create new representations of data
      - The new representation may be easier to understand by humans or other machine learning algorithms
    - Common applications
      - **Dimensionality reduction
        - Accepting a high-dimensional representation of data that contains many features
        - Finding a new way to represent that data
        - Generalize its important features with fewer features
        - Common applications
          - **Reducing data to two dimensions (for visualization purposes)
      - Finding the components that "make up" the data
        - Common applications
          - **Extracting themes from a collection of text documents
            - Tasks
              - Find the unknown themes discussed in each document
              - Learning what topics appear in each document
            - Used to track topic discussions on social media
  - **Clustering
    - Split the data into groups
    - Each group contains similar items
    - Common applications
      - **Intelligent categorization of photo albums**
        - Extract all faces
        - Sort similar looking faces into groups.



# 2. Challenges of unsupervised learning

- **Main challenge: assessing whether the algorithm has learned something useful**
- Unsupervised learning algorithms are generally used on data that does not contain any labeling information, so we don't know what the correct output should be.
- We have no way to "tell" the algorithm what we want.
- We don't have a way to "tell" the algorithm what we want Typically, the only way to evaluate the results of unsupervised algorithms is to check them manually
- Unsupervised algorithms can often be used for exploratory purposes if the data scientist wants to understand the data better, rather than as part of a large automated system.
- **Another common application of unsupervised algorithms is as a preprocessing step for supervised algorithms **
  - Can improve the accuracy of supervised algorithms
  - Can reduce memory footprint and time overhead

- Simple feature-by-feature scaling and shifting of data

- Left: dichotomized data with two features
  - First eigenvalue: 10~15
  - Second eigenvalue: 1~9
- Right side: 4 data transformation methods
  - **StandardScaler**
    - Ensure that each feature has a mean of 0 and a variance of 1 so that all features are of the same magnitude
    - Does not ensure any particular maximum and minimum values for features
  - **RobustScaler**
    - Ensure that the statistical properties of each feature lie in the same range
      - Median and quartiles
    - Ignores data points that are very different from others (**Outliers**)
  - **MinMaxScaler**
    - Make all features lie exactly in the range 0 to 1
  - **Normalizer**
    - Scale each data point so that the Euclidean length of the feature vector is equal to 1
      - Project each data point onto a circle (sphere) of radius 1
    - Scaling is different for each data point

## 3.3 Perform the same scaling on both training and test data

```python
import mglearn

from sklearn.preprocessing import MinMaxScaler

# 将正确缩放的数据可视化
```

## 3.4 The role of preprocessing for supervised learning

```python
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split
from sklearn.svm import SVC
from sklearn.preprocessing import MinMaxScaler
from sklearn.preprocessing import StandardScaler

cancer = load_breast_cancer()
X_train, X_test, y_train, y_test = train_test_split(cancer.data, cancer.target, random_state=0)

svm = SVC(C=100)
svm.fit(X_train, y_train)

print("test score: {:.3f}".format(svm.score(X_test, y_test)))
# test score: 0.944

# 使用0-1缩放进行预处理
scaler = MinMaxScaler()
scaler.fit(X_train)
X_train_scaled = scaler.transform(X_train)
X_test_scaled = scaler.transform(X_test)

# 在缩放后的训练数据上学习SVM
svm.fit(X_train_scaled, y_train)

# 在缩放后的测试集上计算分数
print("test score: {:.3f}".format(svm.score(X_test_scaled, y_test)))
# test score: 0.965

# 利用零均值和单位方差的缩放方法进行预处理
scaler = StandardScaler()
scaler.fit(X_train)
X_train_scaled = scaler.transform(X_train)
X_test_scaled = scaler.transform(X_test)

# 在缩放后的训练数据上学习SVM
svm.fit(X_train_scaled, y_train)

# 在缩放后的测试集上计算分数
print("test score: {:.3f}".format(svm.score(X_test_scaled, y_test)))
# test score: 0.958
```

## 4.1 Principal Component Analysis (PCA)

- A method of rotating a data set
  - The rotated features are statistically uncorrelated
  - After rotation a subset of the new features is usually chosen based on its importance in explaining the data

```python
import matplotlib.pyplot as plt
import mglearn

mglearn.plots.plot_pca_illustration()

plt.tight_layout()
plt.show()
```

<img src="Chapter 3 Unsupervised Learning and Preprocessing.assets/Data Transformation with PCA.png" alt="Data Transformation with PCA" style="zoom: 50%;" />

- Top left: original data points

  1. the algorithm looks for the direction with the highest variance (Component 1)
     - The direction that contains the most information in the data

  2. the algorithm finds the direction that is orthogonal to the first direction and contains the most information

  - The direction found using this method is called the **principal component**.
    - The principal directions of the variance of the data
  - The number of principal components is the same as the original features

- Top right: rotate the original data so that the first principal component is parallel to the x-axis and the second principal component is parallel to the y-axis

  - Before rotation, the data is subtracted from the mean
    - Before rotation, the data is subtracted from the mean value so that the transformed data is centered on 0

- Bottom left: only the first principal component is retained

  - Reducing two-dimensional data to one-dimensional data

- Bottom right: reverse rotation and re-adding the mean to the data

  - Remove the effects of noise from the data
  - Visualize the portion of information retained in the principal components.

     from sklearn.preprocessing import StandardScaler

## 4.3 Streamform learning with t-SNE

- Streamform learning algorithm

  - Algorithm for visualization
  - Allows for complex mappings
  - Can give good visualization
  - Algorithm computes a new representation of the training data, but does not allow transforming the new data
  - **Only transform data used for testing**

- t-SNE

  - Idea: find a 2D representation of the data that maintains the distance between data points as much as possible
  - Steps

  - **Focus on points that are closer together***
  - Try to preserve information that indicates which points are closer together
  - **Can clearly separate categories based only on proximity between data points in the original space

# 5. clustering

- The task of dividing a data set into groups (**clusters**)
- Goal: divide the data so that data points within a cluster are very similar and data points within different clusters are very different
- The algorithm assigns (or predicts) a number to each data point indicating which cluster the point belongs to



## 5.1 k-mean clustering

- Attempts to find **cluster centers** that represent specific regions of the data

- Steps

  1. Assign each data point to the nearest cluster center

  2. set each cluster center to the average of all assigned data points

  3. repeat the above two steps until the cluster assignments do not change again

- Algorithm Description


  <img src="Chapter 3 Unsupervised Learning and Preprocessing.assets/Input Data and the Three Steps of the k-means Algorithm.png" alt="Input Data and the Three Steps of the k-means Algorithm" style="zoom: 67%;" />

  - Triangle: cluster centers
  - Circle: data points
  - Color: cluster members
  - Finding 3 clusters
    - Initialize the algorithm by declaring 3 random data points as cluster centers.
    - Run the iterative algorithm.
      - Each data point is assigned to the closest cluster center.
      - Modify the cluster center to the average of the assigned points.
    - Repeat 2 times, after the third iteration, the data points assigned to the cluster centers remain the same and the algorithm ends

### 5.1.1 Failure of k-means

- Each cluster is defined only by its center

  - Each cluster is **convex**

- k-means can only find relatively simple shapes

- The **k-mean assumes that all clusters have the same diameter to some extent, and always draws the boundary between clusters just in the middle of the cluster center

### 5.1.3 Advantages, disadvantages

- Advantages
  - Very popular clustering algorithm
  - Relatively easy to understand and implement
  - Relatively fast to run
  - Easily scalable to large datasets
- Disadvantages
  - Relies on random initialization
    - Output of the algorithm depends on random seeds
    - By default, scikit-learn runs the algorithm 10 times with 10 different random initializations and returns the best result (the sum of the variances of the clusters is minimized)
  - Strong constraints on assumptions about the shape of the clusters
  - Requires specifying the number of clusters to be found (which may not be known in real-world applications)



## 5.2 Coalescent clustering

- Many clustering algorithms constructed on the same principles

  - Principle: algorithm first declares each point to be its own cluster, then merges the two most similar clusters until some stopping criterion is satisfied
    - Criterion
      - scikit-learn: number of clusters
    - **link** criterion: specify how to measure the most similar clusters
      - Defined between two existing clusters
      - Three options implemented in scikit-learn
        - **ward**
          - **default option**
          - Pick two clusters to merge so that the increase in variance is minimized across all clusters
          - will result in clusters of about equal size
          - **Used for most datasets
        - **average**
          - Combines the two clusters with the smallest average distance between all points in the cluster.
        - **complete**
          - Combines the two clusters with the smallest maximum distance between points in the cluster.

## 5.3 DBSCAN

- Advantages

  - Does not require the user to set the number of clusters **a priori**.
  - Can divide clusters with complex shapes
  - Can find points that do not belong to any cluster
  - Scalable to relatively large datasets

- Disadvantages

  - Slower operation speed

- **Principle: Recognize points in a "crowded" region of the feature space

  - "Crowded" region (**dense** region): many data points in the region close together
    - Points in a dense region: **core samples**
      - If there are at least min_samples of data points within a distance of eps from a given data point, then that point is a core sample
      - DBSCAN puts core samples that are less than eps away from each other into the same cluster

- **Idea: clusters form dense regions of data, separated by relatively empty regions**

- Steps

  1. select any point
  2. find all the points whose distance to this point is less than or equal to eps
     - If the number of data points within eps of the start point is less than min_samples, the point is labeled as **noise**.
       - This point does not belong to any cluster
     - If the number of data points whose distance from the start point is within eps is greater than min_samples, this point is labeled as a core sample and assigned a new cluster label
  3. visit all neighbors of the point (within distance eps)
     - If they are not already assigned a cluster, assign the new cluster label just created to them
     - If they are core samples, then visit their neighbors in turn
  4. gradually increase the size of the cluster until there are no more core samples within eps distance of the cluster
  5. another unvisited point is selected and the above steps are repeated

## 5.4 Comparison and Evaluation of Clustering Algorithms

## 5.4.1 Evaluating Clustering with True Values

- ** Metrics used to evaluate clustering algorithms relative to true clustering results **

  - **Adjusted rand index (ARI)**
    - Optimal value: 1
    - Irrelevant: 0
  - **Normalized Mutual Information (NMI)**
    - Optimal value: 1
    - Not relevant: 0

- Comparison of k-means, coalescent clustering and DBSCAN algorithms using ARI

  ```python
  import numpy as np
  import mglearn
  from matplotlib import pyplot as plt
  from sklearn.cluster import DBSCAN, KMeans, AgglomerativeClustering
  from sklearn.datasets import make_moons
  from sklearn.preprocessing import StandardScaler
  from sklearn.metrics.cluster import adjusted_rand_score
  
  X, y = make_moons(n_samples=200, noise=0.05, random_state=0)
  
  # 将数据缩放成平均值为0、方差为1
  scaler = StandardScaler()
  scaler.fit(X)
  X_scaled = scaler.transform(X)
  
  fig, axes = plt.subplots(1, 4, figsize=(15, 3), subplot_kw={'xticks': (), 'yticks': ()})
  
  # 列出要使用的算法
  algorithms = [KMeans(n_clusters=2), AgglomerativeClustering(n_clusters=2), DBSCAN()]
  
  # 创建一个随机的簇分配，作为参考
  random_state = np.random.RandomState(seed=0)
  random_clusters = random_state.randint(low=0, high=2, size=len(X))
  
  # 绘制随机分配
  axes[0].scatter(X_scaled[:, 0], X_scaled[:, 1], c=random_clusters, cmap=mglearn.cm3, s=60)
  axes[0].set_title("Random assignment - ARI: {:.2f}".format(adjusted_rand_score(y, random_clusters)))
  
  for ax, algorithm in zip(axes[1:], algorithms):
      # 绘制簇分配和簇中心
      clusters = algorithm.fit_predict(X_scaled)
      ax.scatter(X_scaled[:, 0], X_scaled[:, 1], c=clusters, cmap=mglearn.cm3, s=60)
      ax.set_title("{} - ARI: {:.2f}".format(algorithm.__class__.__name__, adjusted_rand_score(y, clusters)))
  
  plt.tight_layout()
  plt.show()
  ```

## 5.5 Summary of clustering methods

- The application and evaluation of clustering is a very qualitative process and is usually helpful in the exploratory phase of data analysis
- Three clustering algorithms
  - k-means
    - Allows the number of desired clusters to be specified
    - Clusters can be represented by the mean of the clusters
    - Can be viewed as a decomposition method where each data point is represented by its cluster center
  - DBSCAN
    - Allows to define the proximity with the eps parameter, which indirectly affects the size of the clusters.
    - Can detect "noise points" that are not assigned any clusters.
    - Can help to automatically determine the number of clusters
    - Allows clusters to have complex shapes
  - Coalescent clustering
    - Allows the number of desired clusters to be specified
    - Provides the entire hierarchy of possible divisions of the data.
    - Can be easily viewed through a tree diagram
- All three algorithms can control the **granularity** of the clustering
- All three methods can be used on large real-world datasets, all are relatively easy to understand, and all can cluster into multiple clusters
