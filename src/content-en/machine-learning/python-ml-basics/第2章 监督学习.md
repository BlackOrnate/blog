# 1. Classification and regression

- Types of Supervised Machine Learning Problems

  - **Categorization**
    - Goal: Predict **category labels**
      - These labels come from a predefined optional list
    - Example.
      - Iris is categorized into one of three possible species
    - **Dual categorization**
      - **Positive category**: object of study
      - **Anti-classification
    - **Multi-classification
  - **Regression
    - Objective: predict a continuous value (floating point, real)
- Distinguish between categorization and regression: **whether the output has some kind of continuity**

  - With continuity: regression problem
  - Without continuity: classification problems

# 2. Generalization, overfitting and underfitting

- If a model can make accurate predictions about data it hasn't seen before, then it can generalize** from the training set** to the test set
- **Simple models are better at generalizing to new data***
- If one pays too much attention to the details of the training set when fitting the model and gets a model that performs well on the training set but does not generalize to new data, then there is **overfitting**
- If the model is too simple and performs poorly on the training set, then there is **underfitting**
- Tradeoff between overfitting and underfitting

  ! [Tradeoffs between model complexity and training and testing accuracy](https://i-blog.csdnimg.cn/blog_migrate/515e93248a1c356c76b45d229e1823a3.png#pic_center)

## Model complexity versus dataset size

- Model complexity is closely related to the variability of the inputs in the dataset: the greater the range of variability of the data points contained in the dataset, the more complex the model can be used without overfitting
- Collecting more data points allows for a greater range of variation, so larger data sets can be used to build more complex models

  training_accuracy = []
  test_accuracy = []

  # [1,10]

      clf = KNeighborsClassifier(n_neighbors=n_neighbors)
      # 构建模型
      clf.fit(X_train, y_train)
      
      training_accuracy.append(clf.score(X_train, y_train))
      # 记录训练集精度
      
      test_accuracy.append(clf.score(X_test, y_test))
      # 记录测试集精度
  
  plt.plot(neighbors_settings, training_accuracy, label="training accuracy")
  plt.plot(neighbors_settings, test_accuracy, label="training accuracy")
  plt.xlabel("n_neighbors")
  plt.ylabel("Accuracy")
  plt.legend()
  plt.show()

### 3.2.4 Analyzing the KNeighborsRegressor

```python
import numpy as np
import matplotlib.pyplot as plt
import mglearn
from sklearn.neighbors import KNeighborsRegressor
from sklearn.model_selection import train_test_split

X, y = mglearn.datasets.make_wave(n_samples=40)
X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=0)

fig, axes = plt.subplots(1, 3, figsize=(15, 4))

line = np.linspace(-3, 3, 1000).reshape(-1, 1)
# 在[-3,3]之间均匀分布的1000个数

for n_neighbors, ax in zip([1, 3, 9], axes):
    reg = KNeighborsRegressor(n_neighbors=n_neighbors)
    reg.fit(X_train, y_train)
    ax.plot(line, reg.predict(line))
    ax.plot(X_train, y_train, "^", c=mglearn.cm2(0), markersize=8)
    ax.plot(X_test, y_test, "v", c=mglearn.cm2(1), markersize=8)
    ax.set_title(f"{n_neighbors} neighbor(s) \n train score {reg.score(X_train, y_train):.2f} test score {reg.score(X_test, y_test):.2f}") 
    ax.set_xlabel("feature")
    ax.set_ylabel("target")

axes[0].legend(["predictions", "training", "test"], loc="best")
plt.tight_layout()
plt.show()
```

- Using only a single neighbor
  - Every point in the training set has a significant effect on the prediction result
  - The image of the prediction result passes through all data points
  - Leads to very unstable prediction results

  - The fit to the training data is also poor.

### 3.2.5 Advantages, disadvantages and parameters

- The KNeighbors classifier has 2 important parameters
  - Number of neighbors
    - Using a smaller number of neighbors tends to give better results
  - Measure of distance between data points
    - Euclidean distance is used by default
- Advantages.
  - The model is easy to understand and usually does not require much tuning to get good performance.
- Disadvantages
  - Predictions can be slow if the training set is large
  - Doesn't work well for datasets with many features
  - Especially bad for datasets where most of the features have most of their values as 0 (sparse datasets).

## 3.3 Linear Models

## 3.3.1 Linear models for regression

- General formula for linear model prediction

  $$
  \hat{y} = w[0]*x[0] + w[1]*x[1] + \cdots + w[p]*x[p] + b
  $$

  - $x[p]$: characterization of individual data points
  - $w$: parameters
  - $b$: parameters
  - $\hat{y}$.
- Data set formula for a single feature

  $$
  \hat{y} = w[0]*x[0] + b
  $$.

### 3.3.2 Linear regression (ordinary least squares) (sklearn.linear\_model.LinearRegression)

- Find the parameters w and b such that the **mean square error** between the predicted value of the training set and the true regression target value y is minimized
  - Mean square error: the sum of the squares of the differences between the predicted and true values divided by the number of samples
- **No parameters
- **No control over the complexity of the model**

```python
import mglearn
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import train_test_split

X, y = mglearn.datasets.make_wave(n_samples=60)
X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=42)

lr = LinearRegression().fit(X_train, y_train)

print(f"{lr.coef_}")
# [0.39390555]
# coef_: 斜率,系数,权重
# Numpy数组

print(f"{lr.intercept_}")
# -0.03180434302675973
# intercept_: 截距
# 浮点数

print(f"training score: {lr.score(X_train, y_train):.2f}")
# training score: 0.67

print(f"test score: {lr.score(X_test, y_test):.2f}")
# test score: 0.66
```

- Close scores on the training and test sets and not very good results indicate **underfitting**

```python

### 3.3.3 Ridge regression (sklearn.linear\_model.Ridge)

- The choice of w should not only give good results on the training data, but also fit additional constraints (**regularization**)
  - All element values of w should be close to 0
- **Use of L2 regularization**

```python
import mglearn
from sklearn.model_selection import train_test_split
from sklearn.linear_model import Ridge

X, y = mglearn.datasets.load_extended_boston()
X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=0)

ridge = Ridge().fit(X_train, y_train)

print(f"training score: {ridge.score(X_train, y_train):.2f}")
# training score: 0.89

print(f"test score: {ridge.score(X_test, y_test):.2f}")
# test score: 0.75
```.

- The Ridge model makes a trade-off between model simplicity and training set performance
- alpha parameter: how important simplicity and training set performance are to the model

  - Increase alpha: coefficients converge more towards 0, thus decreasing training set performance and increasing generalization performance

    ```python
    import mglearn
    from sklearn.model_selection import train_test_split
    from sklearn.linear_model import Ridge
    
    X, y = mglearn.datasets.load_extended_boston()
    X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=0)
    
    ridge10 = Ridge(alpha=10).fit(X_train, y_train)
    
    print(f"training score: {ridge10.score(X_train, y_train):.2f}")
    # training score: 0.79
    
    print(f"test score: {ridge10.score(X_test, y_test):.2f}")
    # test score: 0.64
    ```

### 3.3.5 Linear Models for Classification

- Prediction formula for binary classification

  $$
  \hat{y} = w[0]*x[0] + w[1]*x[1] + \cdots + w[p]*x[p] + b > 0
  $$

  - If the function value is less than 0, the prediction category is -1
  - If the function value is greater than 0, the prediction category is +1
- Linear model used for regression, output $\hat{y}$
- Linear model used for classification, **decision boundary** is a linear function of the inputs

  - Binary linear classifiers are classifiers that use lines, planes, or hyperplanes to separate categories
- Learning the difference between linear modeling algorithms

  - Specific combinations of coefficients and intercepts **metrics of how well **they fit the training data
    - **loss function**
  - Whether to use regularization and which regularization to use
- Common linear classification algorithms

  - **Logistic regression (sklearn.linear\_model.LogisticRegression)
  - **Linear Support Vector Machines (sklearn.svm.LinearSVC)

### 3.3.6 Linear models for multiclassification

- Many linear classification models are only applicable to binary classification problems

  - Many linear classification models are only applicable to binary classification problems, except for logistic regression.
- Common methods for generalizing dichotomous algorithms to multiclassification algorithms: **pair of the rest**

  - Each category learns a binary classification model that separates this category from all other categories as much as possible.
  - Run all binary classifiers on test points to make predictions
  - The classifier with the highest score on the corresponding category "wins", and the category label is returned as the prediction.

    - Classification Confidence Equation

      Classification Confidence Equation $$
      w[0]*x[0] + w[1]*x[1] + \cdots + w[p]*x[p] + b
      $$
- Triple classification

### 3.3.7 Advantages, disadvantages and parameters

- Main Parameters

  - Regularization parameters
    - Regression model: alpha
    - Classification model: C
    - **The larger the alpha or the smaller the C, the simpler the model (the better the generalization) **
    - Usually search for alpha and C on a logarithmic scale
  - Choice of regularization
    - L1 regularization (lasso)
      - Wish to keep only a few important features
      - Want to have high interpretability of the model
    - L2 regularization (ridge regression, logistic regression, linear support vector machines)
- Advantages

  - Fast training
  - Fast prediction
  - Suitable for large datasets as well as sparse datasets
    - Large datasets: accelerated by using the solver='sag' option in ridge regression and logistic regression models
  - Easy to understand the prediction process
  - Suitable for datasets with more features than samples
- Disadvantages

  - Difficult to interpret coefficients when the dataset contains highly correlated features
  - Other models have better generalization ability in lower dimensional spaces

## 3.4 Plain Bayesian Classifier

- Very similar to linear models.
- **Faster to train***

  - Learn parameters by looking at each feature individually
  - Collect simple category statistics from each feature
- **Poor generalization ability (compared to Logistic Regression and Linear Support Vector Machines)
- 3 Plain Bayesian classifiers

  - **GaussianNB (sklearn.naive\_bayes.)

    - Applied to arbitrary continuous data
    - **Saves the mean and standard deviation of each feature in each category
  - **BernoulliNB (sklearn.naive\_bayes.BernoulliNB)

    - Assuming the input data is binary categorized
    - Mainly used for text data categorization
    - **Calculate the number of elements in each category for which each feature is not zero

    ```python
    import numpy as np
    
    X = np.array([[0, 1, 0, 1],
                  [1, 0, 1, 1],
                  [0, 0, 0, 1],
                  [1, 0, 1, 0]])
    y = np.array([0, 1, 0, 1])
    counts = {}
    
    for label in np.unique(y):
        counts[label] = X[y == label].sum(axis=0)
    
    print(f"counts: {counts}")
    # counts: {0: array([0, 1, 0, 2]), 1: array([2, 0, 2, 1])}
    ```

    - The prediction formula is identical to the linear model
  - **MultinomialNB (sklearn.naive\_bayes.)

    - Assumes input data is count data
    - Mainly used for text data categorization
    - **Calculate the mean value of each feature in each category**
    - The prediction formula is identical to the linear model

  - Higher impact on accuracy
- Application
  - High-dimensional data

    - BernoulliNB

  - Fast training

### 3.5.1 Constructing a decision tree

- Dataset

  - Each category contains 50 data points
- The if/else problem is called a **test**
- The algorithm traverses all possible tests to find the one that is most informative for the target variable
- Steps to generate a decision tree

  - 1st test

    ! [Decision boundary of tree with depth 1 (left) with corresponding tree (right)](https://i-blog.csdnimg.cn/blog_migrate/d861fd881f25a95f6777c1decb2e161d.jpeg#pic_center)

    - Putting the dataset at x[1] = 0.0596 maximizes the distinction between points in category 0 and points in category 1
    - **Root node** represents the entire data set
    - The left node corresponds to the top region
    - The right node corresponds to the bottom region
  - 2nd test

    ! [Decision boundary of tree of depth 2 (left) with corresponding tree (right)](https://i-blog.csdnimg.cn/blog_migrate/51aaab2bb0002352e8dd107a8b6f526b.jpeg#pic_center)

    - Make a distinction based on x[0]
  - Repeat the recursive division
  - 4th test

    ! [Decision boundary for tree with depth 9 (left) and corresponding tree (right)](https://i-blog.csdnimg.cn/blog_migrate/69f566ce768e50ea27c5892b9db4ba67.jpeg#pic_center)

    - A leaf node is **pure** if all the data points it contains have the same target value
- Steps for predicting new data points

  - Classification
    - Check which region of the feature space the point is in.
    - Use most of the target values in that region as predictions
    - Traverse the tree from the root node to find this region.
  - Regression task
    - Traverse the tree based on tests at each node.
    - Find the leaf node to which the new data band point belongs.
    - Output: average target value for all training points in this leaf node

### 3.5.2 Controlling the complexity of decision trees

- **Constructing a decision tree until all leaf nodes are pure leaf nodes leads to a very complex model and is highly overfitted to the training data **
- Strategies to prevent overfitting

  - **Pre-pruning**: stopping tree growth early
    - Restrictions
      - Maximum depth of the tree
      - Maximum number of leaf nodes
      - Minimum number of data points in a node
  - **Post pruning**: construct the tree first, then remove or collapse nodes with little information
- 2 types of classification decision trees

  - **DecisionTreeRegressor (sklearn.tree.)
  - **DecisionTreeClassifier (sklearn.tree.DecisionTreeClassifier)
- **sklearn only implements pre-pruning **
- Performs a full unfolding of the tree

  - The tree keeps branching until all leaf nodes are pure

  ```python
  from sklearn.datasets import load_breast_cancer
  from sklearn.model_selection import train_test_split
  from sklearn.tree import DecisionTreeClassifier
  
  cancer = load_breast_cancer()
  X_train, X_test, y_train, y_test = train_test_split(cancer.data, cancer.target, stratify=cancer.target, random_state=42)
  
  tree = DecisionTreeClassifier(random_state=0)
  tree.fit(X_train, y_train)
  
  print(f"training score: {tree.score(X_train, y_train):.3f}")
  # training score: 1.000

## 3.6 Decision Tree Integration

- **Integration**: a method of merging multiple machine learning models to build more powerful models
- There are two integration models that are effective for large datasets for both classification and regression
  - **Random forests
  - **Gradient Boosting Decision Trees

### 3.6.1 Random Forests

- **Essence**: a collection of many decision trees, where each tree is slightly different from the others
- **Idea**: each tree can be overfitted, if many trees are constructed, the results of these trees can be averaged to reduce overfitting
- **adds randomness to the tree construction process to ensure that each tree is different
- Randomization methods for trees

  - Select the data points used to construct the tree
  - Select features for each segmentation test
- Use categorization

  - **Classification: sklearn.ensemble.RandomForestClassifier**
  - **regression: sklearn.ensemble.RandomForestRegressor**

#### Constructing a Random Forest

- Steps
  - Determine the number of trees to be used for construction
    - Parameters: **n\\_estimators**
  - Constructing a tree
    - Perform **self-sampling** of the data to ensure that the trees are distinct from each other
      - Repeatedly randomize a sample from the **n\_samples** data points with putback, repeat **n\_samples** times
    - Construct a decision tree based on this newly created data set
      - At each node, randomly select a subset of features and find the best test for one of them
        - Do not do find the best test at each node
        - Number of features: **max\_features**
          - If max\_features=n\_features, then all features of the dataset are considered for each division
          - If max\_features=1, then there is no way to choose which feature to test on when dividing, only to search for a different threshold for a randomly selected feature
          - **If max\_features is large, then the trees in the random forest will be very similar and the data can be easily fitted using the most unique features**
          - **If max\_features is small, then the trees in the random forest will be very different, and the depth of each tree will have to be large in order to fit the data well***
- For regression problems, these results can be averaged as the final prediction.
- For classification problems, a "soft voting" strategy is used.

#### Advantages, disadvantages and parameters

- Parameters

  - n\_jobs: regulates the number of cores used.
    - Using more CPU cores allows for a linear increase in speed
    - You can set n\_jobs=-1 to use all cores of the computer.
  - random\_state: random state
    - Different random states can completely change the model being built
      - The more trees in the forest, the more robust it is to random state selection
  - n\_estimators: number of trees used for construction
    - The larger the better.
    - Averaging over more trees reduces overfitting and results in a more robust integration
    - But the benefits are diminishing, and the more trees you have the more memory you need and the longer it takes to train.
    - **as many as your time/memory permits** **as many as your time/memory permits
  - max\_features: number of features.
    - **The smaller the value, the lower the overfitting **.
    - For classification, the default value is max\_features=sqrt(n\_features)
    - For regression, the default value is max\_features=n\_feature.
    - Increasing max\_features or max\_leaf\_nodes can sometimes improve performance, and can also greatly reduce the time and space requirements for training and prediction
- Advantages

  - **One of the most widely used machine learning methods today***.
  - Does not require iterative tuning of parameters to give good results
  - No need to scale the data
  - One reason for still using decision trees is the need for a compact representation of the decision process.
    - The depth of the tree in a random forest is often greater than a decision tree (because of the subset of features used)
  - Random forests usually perform well even with very large datasets, and the training process can be easily parallelized on multiple CPU cores of powerful computers
- Disadvantages

  - Often does not perform well on sparse data (text data) with very high dimensionality
    - It may be more appropriate to use a linear model
  - Requires a lot of memory and is slower to train and predict than linear models

### 3.6.2 Gradient boosted regression trees (gradient boosters)

- ** Combines multiple decision trees to build a more powerful model **
- ** constructs trees in a sequential fashion, each tree attempts to correct the errors of the previous tree **
- By default, there is no randomization, but strong pre-pruning
- **usually use trees with very small depths (1 to 5)
- **idea**: merge many simple models (trees with small depth)
  - Each tree can only make good predictions for part of the data
  - The more trees you add, the stronger the performance will be
- Using classification
  - **classification: sklearn.ensemble.GradientBoostingClassifier**
  - **Regression: sklearn.ensemble.GradientBoostingRegressor**
- **Sensitive to parameter settings**
  - Pre-Pruning
  - Number of trees: n\_estimators
  - Learning rate: learning\_rate
    - Controls the strength of each tree to correct the errors of the previous tree

### 3.7.2 Kernel tricks

- **Principle**: directly compute the distance between data points in the extended feature representation (inner product) without actually computing the extension
- Common methods for mapping data into higher dimensional spaces

  - **Polynomial kernel**: computes all possible polynomials of the original feature up to a certain order
  - **Radial Basis Function Kernel (Gaussian Kernel, RBF Kernel)**: consider all possible polynomials of all orders, but the higher the order, the less important the feature is

### 3.7.3 Understanding SVMs

- Usually only a subset of training data points are important for defining decision boundaries: those points that lie on the boundary between classes (**support vectors**)
- **Categorization decisions**: made based on the distance between the new sample points and the support vectors and the importance of the support vectors learned during training (dual\_coef\_)

  - Distance between data points (given by Gaussian kernel)

    $$
    k_{rbf} (x_1, x_2) = exp(-\gamma \| x_1 - x_2 \|^2)
    $$.

    - $x_1$ and $x_2$: data points
    - $\| x_1 - x_2 \|$: Euclidean distance
    - $\gamma$: parameters controlling the width of the Gaussian kernel

```python

### 3.7.4 SVM tuning parameters

- gamma: controls the width of the Gaussian kernel
  - Determines how close the points are to each other.
  - Small gamma.
    - Decision boundaries change slowly.
    - Generate models with low complexity
  - Large gamma.
    - Large gamma. Decision boundaries change quickly.
    - Generate models with higher complexity
- C: Regularization parameters
  - Limit the importance of each point (dual\_coef\_)
  - Small C

### 3.8.1 Neural network modeling

- MLPs can be considered as generalized linear models

- The prediction formula for the linear model
  $$
  \hat{y} = w[0]*x[0] + w[1]*x[1] + \cdots + w[p]*x[p] + b
  $$

  - $\hat{y}$: weighted summation of the input features $x[0]$ to $x[p]$.

- Weights: learned coefficients $w[0]$ to $w[p]$.

- Visualization of formulas

    ```python
    import mglearn
    
    mglearn.plots.plot_logistic_regression_graph()
  ```

  ! [Visualization of Logistic Regression](https://i-blog.csdnimg.cn/blog_migrate/41b3f87b4072efc41683e4a15929cb28.png#pic_center)

  - Each node on the left: an input feature
  - Connected: learned coefficients
  - Node on the right: output (weighted sum of inputs)
- MLP repeats the computation of the weighted summation several times

  - Steps
    - Calculate the **hidden units** representing the intermediate processes.
    - Calculate the weighted sum of these hidden units
    - The final result is obtained

  ```python
  import mglearn
  
  mglearn.plots.plot_single_hidden_layer_graph()
  ```

  ! [Illustration of a multilayer perceptron with a single hidden layer](https://i-blog.csdnimg.cn/blog_migrate/51b0756e051528bb9f6466ccc0a2cbef.png#pic_center)

  - Need to learn more coefficients (weights)
    - One coefficient between each input and each hidden unit
    - One coefficient between each hidden unit and the output
- In order for MPL to be more powerful than a linear model, a trick is needed

  - After calculating the weighted sum for each hidden cell, apply another **nonlinear function** to the result
    - **corrected nonlinear (relu) **
    - **Tangent hyperbola (tanh) **
  - The result of this function is used for weighted summation, and the output is computed
    $$
    \hat{y}
    $$

  - relu
    - Truncate values less than 0
  - tanh
    - For small inputs: close to -1
    - For large inputs: close to +1
- For small neural networks (multi-layer perceptron with a single hidden layer), the full formula for $\hat{y}$ to compute the regression problem (using tanh nonlinearity)
  $$
  h[0] = tanh(w[0,0]*x[0] + w[1,0]*x[1] + w[2,0]*x[2] + w[3,0]*x[3] + b[0]) \\
  h[1] = tanh(w[0,0]*x[0] + w[1,0]*x[1] + w[2,0]*x[2] + w[3,0]*x[3] + b[1]) \\
  h[2] = tanh(w[0,0]*x[0] + w[1,0]*x[1] + w[2,0]*x[2] + w[3,0]*x[3] + b[2]) \\
  \hat{y} = v[0]*h[0] + v[1]*h[1] + v[2]*h[2] + b
  $$

  - $w$: weights between input $x$ and hidden layer $h$
  - $v$: weights between hidden layer $h$ and output $\hat{y}$
  - Weights $w$ and $v$ to be learned from the data
  - $x$: input features
  - $\hat{y}$: computed outputs
  - $h$: intermediate result of the computation
- **Number of nodes in the hidden layer: an important parameter that needs to be set by the user**

  - For very small or simple datasets: the number of nodes can be very small.
  - For very complex datasets: the number of nodes can be very large.

### 3.8.4 Estimating the complexity of neural networks

- Start with 1 or 2 hidden layers, and then you can gradually increase the
- The number of nodes per hidden layer is usually close to the number of input features, but rarely more than the number of features at several thousand nodes.
- When considering the model complexity of a neural network, a useful metric is **the number of learned weights (or coefficients)**
- If there is a binary classification dataset containing 100 features, the model has 100 hidden units

  - There are then 100\*100=10000 weights between the input layer and the first hidden layer
  - Between the hidden layer and the output layer there are another 100\*1=100 weights
  - The total number of weights is about 10,100.
- If we add a second hidden layer with 100 hidden units.

  - If we add a second hidden layer containing 100 hidden units, there are 100\*100=10000 weights between the first hidden layer and the second hidden layer.
  - The total number of weights becomes about 20,100.
- If we use a single hidden layer with 1000 hidden units.

  - 100\*1000=100000 weights need to be learned between the input layer and the hidden layer
  - Between the hidden layer and the output layer, you need to learn 1000\*1=1000 weights.
  - Total 101000 weights
- If we add a second hidden layer

  - If we add a second hidden layer, it will add 1000\*1000=1000000 weights.
  - The total number of weights becomes a huge 1,101,000 weights.
  - The total becomes a massive 1101000 weights, 50 times larger than a model with 2 hidden layers of 100 units each.
- **Common methods of parameterization***

  - Create a network large enough to **overfit** and make sure this network can learn about the task
  - Enhance regularization by shrinking the network or increasing the alpha (improves generalization performance)
- **How to learn a model (algorithms used to learn parameters)

  - Set by solver parameters
    - solver has two good options
    - The default option is 'adam'
      - Works well in most cases
      - It is quite sensitive to the scaling of the data (so it is important to always scale the data to have a mean of 0 and a variance of 1)
    - Another option is 'lbfgs'
      - Robustness is quite good
      - Will take longer on large models or large datasets

## Summary

- decision\_function and predict\_proba always have the same shape, both (n\_samples, n\_classes)

  - Except for binary classification problems, decision\_function is (n\_samples, 1)
- If there are n\_classes columns, the prediction result can be achieved by calculating the argmax of each column
- **Comparison of the results of predict with those of decision\_function or predict\_proba requires the use of the classes\_property of the classifier to get the real property names**

  ```python
  import numpy as np
  from sklearn.datasets import load_iris
  from sklearn.linear_model import LogisticRegression
  from sklearn.model_selection import train_test_split
  
  iris = load_iris()
  X_train, X_test, y_train, y_test = train_test_split(iris.data, iris.target, random_state=42)
  
  logreg = LogisticRegression()
  named_target = iris.target_names[y_train]
  logreg.fit(X_train, named_target)
  
  print("unique classes in training data: {}".format(logreg.classes_))
  # unique classes in training data: ['setosa' 'versicolor' 'virginica']
  
  print("predictions: {}".format(logreg.predict(X_test)[:10]))
  # predictions: ['versicolor' 'setosa' 'virginica' 'versicolor' 'versicolor' 'setosa' 'versicolor' 'virginica' 'versicolor' 'versicolor']
  
  argmax_dec_func = np.argmax(logreg.decision_function(X_test), axis=1)
  
  print("argmax of decision function: {}".format(argmax_dec_func[:10]))
  # argmax of decision function: [1 0 2 1 1 0 1 2 1 1]
  
  print("argmax combined with classes: {}".format(logreg.classes_[argmax_dec_func][:10]))
  # argmax combined with classes: ['versicolor' 'setosa' 'virginica' 'versicolor' 'versicolor' 'setosa' 'versicolor' 'virginica' 'versicolor' 'versicolor']
