## 1. Object overview

- Superclass, base class, direct or indirect parent of all classes, at the top of the inheritance tree.
- Any class that does not write extends to show that it inherits from a class inherits directly from the Object class, otherwise it inherits indirectly.
- The methods defined in the Object class are the same methods that all objects have.
- The Object type can store any object.
  - As a parameter, it can accept any object.
  - As a return value, it can return any object.

## 2. getClass method

- public final Class<? > getClass(){}
- Returns the actual object type stored in the reference
- Application: Usually used to determine whether the actual object type stored in two references is the same.

```java
public class Test {
    public static void main(String[] args) {
        Student student1 = new Student("student1", 10);
        Student student2 = new Student("student2", 20);

        // 判断student1和student2是否为同一类型
        Class class1 = student1.getClass();
        Class class2 = student2.getClass();
        System.out.println(class1 == class2);
    }
}
```

```java
        this.name = name;
    }

    public int getAge() {

    }
}
```

## 3. hashCode method

- public int hashCode(){}
- Returns the hash code value of this object
- hash value based on the address of the ** object ** or ** string ** or ** number ** using the hash algorithm to calculate the value of the type int
- Generally the same object returns the same hash code

```java
public class Test {
```

## 4. toString method

- public String toString()){}
- Returns a string representation of the object (presentation)
- This method can be overridden according to the needs of the program, e.g., to display the values of the properties of the object

```java
public class Test {
    public static void main(String[] args) {
        Student student1 = new Student("student1", 10);
        Student student2 = new Student("student2", 20);

        System.out.println(student1.toString());
        System.out.println(student2.toString());
    }
}
```

```java
public class Student {
    private String name;
    private int age;

    public Student() {
    }

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {

    }

        this.age = age;
    }

    @Override
    public String toString() {
        return "Student{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }
}
```

## 5. equals method

- public boolean equals(Object obj){}
- Default implementation is (this == obj), comparing two objects with the same address
- Can be overridden to compare if the contents of two objects are the same

```java
public class Test {
    public static void main(String[] args) {
        Student student1 = new Student("student1", 10);
        Student student2 = new Student("student2", 20);
        Student student3 = new Student("student1", 10);

        System.out.println(student1.equals(student2));
        System.out.println(student1.equals(student3));
    }
}
```

```java
public class Student {
    private String name;
    private int age;

    public Student() {
    }

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Student)) return false;
        Student student = (Student) o;
        return age == student.age && name.equals(student.name);
    }
}
```

## 6. finalize method

- This method is automatically called by the JVM when an object is determined to be garbage, and is used to mark the object as garbage and put it into the recycling queue.
- Garbage: when there is no valid reference to the object, it is a garbage object.
- Garbage collection: GC destroys the garbage object and releases the data storage space.
- Automatic recycling mechanism: JVM memory exhaustion, one-time recovery of all garbage objects
- Manual recycling mechanism: use System. gc(); notify JVM to perform garbage collection.
- This method has been removed by JDK

This method has been removed from the JDK.
