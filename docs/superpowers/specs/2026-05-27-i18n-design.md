# i18n 中英文切换 — 设计文档

**日期：** 2026-05-27  
**状态：** 已批准，待实现

---

## 1. 目标

为 BlackOrnate 博客添加完整的中英文双语支持：
- 所有 UI 文字（导航、标题、标签、页脚等）可切换
- 所有 76 篇文章正文可切换（英文版由 OpenAI API 预先翻译生成）
- 语言偏好持久化到 `localStorage`

---

## 3. 翻译脚本（scripts/translate.js）

### 运行方式

```bash
OPENAI_API_KEY=sk-xxx node scripts/translate.js
```

### 行为

1. 遍历 `src/content/` 下所有 `.md` 文件
2. 跳过 `src/content-en/` 中已存在的对应文件（断点续传）
3. 逐篇调用 OpenAI API（模型：`gpt-4o-mini`）
4. 将翻译结果写入 `src/content-en/`，保持目录结构和文件名不变
5. 提取每篇翻译结果的第一个标题，追加到 `src/i18n/post-titles-en.js`
6. 终端实时显示进度：`[12/76] Translating: 第3章 无监督学习...`

### 翻译 Prompt

```
You are a technical translator. Translate the following Chinese Markdown article to English.
Rules:
- Preserve ALL Markdown syntax exactly (headings, code blocks, tables, math, lists)
- Do NOT translate content inside code blocks (``` blocks)
- Do NOT translate LaTeX math expressions ($...$ and $$...$$)
- Translate technical terms accurately
- Keep the same document structure and heading hierarchy
Return only the translated Markdown, no explanation.
```

### 费用估算

76 篇文章 ≈ 40 万汉字，使用 `gpt-4o-mini`，预计一次性费用 **$0.5–1 USD**。

## 4. i18n 系统

### LangContext

```js
// src/contexts/LangContext.jsx
const { lang, t, toggle } = useLang()
// lang: 'zh' | 'en'
// t: 当前语言的字符串对象
// toggle: 切换语言
```

- 初始值从 `localStorage.getItem('lang')` 读取，默认 `'zh'`
- 切换时写回 `localStorage`

### i18n 字符串文件结构

```js
// src/i18n/en.js
export default {
  nav:     { home, columns, about },
  home:    { eyebrow, title, subtitle, recentPosts, allColumns, viewAll },
  stats:   { articles, columns, years, reads },
  post:    { toc, prev, next, backToColumn, pendingBanner, noEnglish },
  search:  { placeholder, empty, hint },
  about:   { role, bio, skills, education, contact, sectionSkills, sectionEdu, sectionContact },
  columns: { /* 每个 columnId 对应 nameEn, descEn */ },
  footer:  { built, articles },
  common:  { loading, notFound },
}
```

### 专栏名称

`src/data/columns.js` 每个专栏条目增加：
```js
{ id: 'python-ml-basics', name: 'Python机器学习基础教程', nameEn: 'Python ML Fundamentals', descEn: '...' }
```

### 文章标题

`src/i18n/post-titles-en.js`（由脚本自动生成）：
```js
export const postTitlesEn = {
  136405772: 'Chapter 2: Linear Algebra',
  135496627: 'Chapter 1: Introduction',
  // ...
}
```

## 5. 语言切换按钮 UI

- 位置：Navbar 右侧，主题切换按钮左边
- 外观：与 `.theme-toggle` 完全一致（32×32，border，hover 金色）
- 文字：当前中文显示 `EN`，当前英文显示 `中`
- 点击调用 `toggle()`

```
[搜索]  [EN]  [🌙]
```

---

## 6. 文章详情页语言切换

- 监听 `lang` 变化，重新调用 `loadPostContent(path, lang)`
- 目录（TOC）随新内容重新提取
- **Fallback**：若 `content-en/` 中对应文件不存在，显示 banner 提示并加载中文版：
  > *"English version not available — showing Chinese"*
- URL 不变，不跳转

---

## 7. 搜索功能

切换为英文时：
- 搜索索引从 `content-en/` 构建（与现有逻辑相同，只换目录）
- `ensureSearchIndex` 根据当前 `lang` 加载对应目录的内容
- 语言切换后索引缓存重置，下次搜索时重新构建

## 8. 功能覆盖范围

| 位置 | 切换内容 |
|------|---------|
| Navbar | 导航项、Logo 标语 |
| 首页 | Hero 区文字、统计标签、专栏名称、文章标题 |
| 专栏列表页 | 专栏名称、描述 |
| 专栏详情页 | 专栏名称、文章标题列表 |
| 文章详情页 | 正文 Markdown、目录、面包屑、前后篇标题 |
| 关于页 | 个人简介、教育背景、技能、章节标题、联系方式标签 |
| 搜索框 | placeholder、提示文字、结果中专栏名 |
| 页脚 | 版权文字、文章数量标签 |

---

## 9. 不在范围内

- 实时翻译（用户输入内容的翻译）
- 第三种语言支持
- URL 路径包含语言前缀（`/en/post/...`）
